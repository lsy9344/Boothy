## 1. 제품 한 줄 정의

Boothy는 `완전 무인 셀프사진관에서 고객이 스스로 촬영을 시작하고, 문제 없이 촬영을 끝내고, 다음 공간으로 자연스럽게 이동하도록 돕는 촬영실 전용 운영 소프트웨어`다.

이 제품의 본질은 사진 편집 툴 자체가 아니다.  
이 제품의 본질은 `무인 매장에서 고객이 막히는 순간을 없애는 운영 자동화 소프트웨어`다.
하드웨어 전제는 `키오스크`가 아니라 `GPU를 갖춘 촬영실용 고사양 데스크톱 PC + 모니터`다.

## 2. 사업 배경

현재 매장은 완전 무인으로 운영된다.

- 고객은 100% 온라인 예약 후 정해진 시간에 방문한다.
- 고객은 스스로 입장하고, 스스로 촬영을 시작하고, 공간을 이동해 사진을 셀렉하고, 인화하고, 집에 간다.
- 운영자가 현장에 상주하지 않기 때문에 고객이 흐름을 멈추는 순간 바로 전화 문의가 발생한다.
- 최근 1개월 기준 전체 지점 합산 전화/원격지원은 약 `500건`으로 추정된다.
- 현재 가장 큰 문제는 `소프트웨어 세팅 및 에러 문제`다.
- 실제 반복 문의는 `촬영 시작 지연`과 `내보내기 지연`에 집중된다.
- 문의 유형 비중의 rough estimate는 `촬영 시작 지연 및 내보내기 지연 70%`, `쿠폰 종료 알림 혼선 20%`, `기타 10%`다.
- 이 `70%` 안에서는 `촬영 시작 지연` 비중이 `내보내기 지연`보다 더 크다.
- 다만 제품 우선순위 관점에서는 `촬영 시작 지연`과 `내보내기 지연`의 중요도를 동일하게 본다.
- `기타 10%`에서 가장 자주 반복되는 항목은 `예약문의`이며, `인원 추가 비용`, `구성`, `옵션` 문의가 중심이다.
- 고객이 실제로 자주 마주하는 대표 오류 문구는 `프로그램 자동화 오류`다.
- 촬영실 입장 직후 사용방법을 몰라 촬영 프로그램을 실행하지 않은 상태에서 카메라 직결 리모컨을 먼저 누르는 상황이 발생한다.
- 이 경우 카메라와 PC의 테더링 연결이 준비되지 않은 상태에서 촬영이 먼저 진행되어, 고객은 촬영했다고 느끼지만 PC에 저장된 사진이 없을 수 있다.
- 촬영 프로그램 실행 전 리모컨 조작은 이후 세팅 병목을 만들고, 카메라 검색/감지 실패로도 이어질 수 있다.
- 내보내기 자동화 오류가 나면 전 타임 고객 사진이 잘못 내보내지거나, 내보내기 지연이 다음 타임 운영 차질로 이어진다.
- 현재 자동화는 라이트룸 클래식 위에서 동작하는 매크로 성격이 강해, 라이트룸의 팝업/메뉴/포커스 상태가 바뀌면 중간 단계에서 멈출 수 있다.
- 별도로 `1시간 추가 쿠폰` 고객이 `n시 50분 촬영 종료 알림`을 보고 시간이 끝난 줄 오해하는 문의가 자주 발생한다.
- 이 연장 고객 정보는 향후 제품이 참조할 `구글시트의 쿠폰 사용 정보`에 기록되어 있다.
- 비즈니스 컨텍스트 기준으로 `평일 1시간 추가 쿠폰` 고객의 운영 시간은 `촬영 1시간 40분 + 내보내기 20분 + 셀렉실 50분`이다.

즉, 이 제품은 단순한 촬영 프로그램 교체가 아니라 아래 문제를 해결하기 위한 사업적 투자다.

- 고객이 촬영 시작 단계에서 막힘
- 촬영 종료 후 사진 변환/전달 단계에서 막힘
- 연장 이용 고객이 자기 세션 시간을 잘못 이해함
- 운영자가 반복적으로 같은 종류의 원격 지원을 수행함
- 무인 매장인데 실제 운영은 "전화 개입형 반자동 매장"처럼 돌아감

### 2.1 현재 운영 환경과 교체 대상

이 PRD는 완전히 새로운 사업을 정의하는 문서가 아니라, 현재 운영 중인 무인 셀프사진관 현장을 기준으로 `촬영실 경험만` 그린필드로 다시 설계하는 문서다.

- 현재 브랜드는 전국 `14개` 매장으로 운영 중이며, 이번 제품의 직접 도입 대상은 오너 관할 `11개` 매장이다.
- 현재 예약 흐름의 기준 시스템은 `네이버 예약`이며, 예약 단위는 `08:00~23:00 정시 시작 1시간 슬롯`이다.
- 현재 고객의 기본 운영 시간은 `촬영 50분 + 내보내기 10분 + 셀렉실 50분`이고, 예외적으로 `1시간 추가 쿠폰` 고객은 `촬영 1시간 40분 + 내보내기 20분 + 셀렉실 50분` 흐름을 가진다.
- 현재 촬영실 현장에서는 `다비 촬영 매니저`와 `다비 내보내기 매니저`가 라이트룸 클래식 자동화 매크로 위에서 동작하고 있다.
- 현재 장애 복구는 주로 `전화 + AnyDesk/Chrome 원격지원`으로 이뤄진다.

즉, Boothy MVP는 현재의 `촬영실 시작/종료 자동화와 예외 대응`을 대체하는 제품이지, 네이버 예약 자체나 셀렉실 편집 경험 전체를 한 번에 교체하는 제품은 아니다.

## 3. 제품 비전

### 3.1 북극성 비전

장기적으로 Boothy는 `무인 셀프사진관 운영 OS`가 되어야 한다.

이 북극성 제품은 다음을 하나의 흐름으로 묶는다.

- 예약과 방문 준비
- 입장과 세션 시작
- 촬영
- 사진 전달
- 셀렉/인화로의 자연스러운 인계
- 오류 감지와 원격 지원
- 운영 로그와 매장별 표준화

즉, 고객이 "혼자서도 끝낼 수 있는 경험"을 만드는 것이 최종 목표다.

### 3.2 이번 MVP의 역할

이번 MVP는 북극성 제품 전체를 한 번에 만들지 않는다.

이번 MVP는 그중에서도 가장 전화가 많이 발생하는 `촬영실 경험`에 집중한다.

이번 MVP의 역할은 다음 3가지다.

1. 고객이 촬영 시작을 스스로 성공하게 만든다.
2. 촬영 중 고객이 불안하지 않게 만든다.
3. 촬영 종료 후 결과물이 셀렉실로 안정적으로 넘어가게 만든다.

## 4. 핵심 문제 정의

### 4.1 고객 관점 문제

고객은 사진을 잘 찍고 싶어서 방문하지, 소프트웨어를 이해하고 싶어서 방문하지 않는다.

현재 고객 관점의 문제는 다음과 같다.

- 어떤 프로그램을 눌러야 하는지 헷갈린다.
- 세팅이 끝났는지, 촬영이 가능한지 확신하기 어렵다.
- 세팅 전에 리모컨을 눌러도 되는지, 지금 눌러도 되는지 판단하기 어렵다.
- 촬영 가능 상태가 표시되기 전에 리모컨을 누르면 촬영은 된 것처럼 보여도 PC에 저장된 사진이 없을 수 있다.
- 어떤 팝업이 떠 있는지, 어떤 화면에서 종료했는지까지 신경 써야 하면 무인 경험이 성립하지 않는다.
- 촬영 중 사진이 잘 들어오고 있는지 불안하다.
- 프리셋이 적용되고 있는지 즉시 확인하고 싶다.
- 촬영 종료 시점에 무엇을 해야 하는지 모르면 곧바로 흐름이 끊긴다.
- 내보내기 오류가 나면 내 사진이 제대로 넘어가는지, 다른 사람 사진과 섞이지 않는지 불안하다.
- 1시간 추가 쿠폰을 사용했는데도 `n시 50분 종료 알림`이 뜨면 남은 시간이 사라진 줄 느낀다.
- 오류가 나면 스스로 복구할 수 없다.

### 4.2 운영자 관점 문제

운영자 관점에서는 같은 유형의 문제가 반복된다.

- 촬영 시작 세팅 실패
- 세팅 전에 리모컨을 눌러 카메라 감지 실패 유발
- 세팅을 완료하지 않은 채 촬영이 진행되어 결과물 문제 발생
- 촬영 종료 후 사진 변환/내보내기 실패
- 내보내기 자동화 오류로 전 타임 사진이 잘못 나가거나 다음 타임이 밀림
- 라이트룸의 예상치 못한 팝업/화면 상태 때문에 매크로가 중간에 멈춤
- 연장 이용 고객이 종료 알림을 보고 시간 차감 또는 오류로 오해해 전화
- 고객이 현재 상태를 이해하지 못해 전화
- 고객이 해도 되는 조작과 하면 안 되는 조작의 경계가 불분명
- 매장마다 같은 운영 정책을 강제하기 어려움

### 4.3 사업 관점 문제

사업적으로는 아래 손실이 발생한다.

- 무인 운영 효율 저하
- 운영자 시간 낭비
- 고객 경험 불안정
- 세팅 미실행 상태에서 촬영이 먼저 진행되면, 고객은 촬영했다고 믿지만 저장된 사진이 없어 결과물을 받지 못하는 치명적 실패가 발생
- 전 타임/현 타임 결과물 혼선은 신뢰 훼손으로 직결됨
- 추가 상품을 구매한 고객이 오히려 더 헷갈리면 상품 신뢰가 떨어짐
- 지점 확장 시 운영 품질 편차 확대
- 반복되는 전화 응대로 인해 브랜드 신뢰도 저하

## 5. 제품 목표

### 5.1 1차 목표

- [x] 고객이 운영자 도움 없이 촬영을 시작할 수 있게 만든다.
- [x] 고객이 촬영 중 "정상 동작 여부"를 스스로 확인할 수 있게 만든다.
- [x] 고객이 촬영 종료 후 다음 단계로 자연스럽게 이동하게 만든다.
- [x] 운영자의 전화/원격지원 건수를 줄인다.

### 5.2 정량 목표

- [x] 기준선은 최근 1개월 전체 지점 합산 약 `500건`이다.
- [x] 도입 후 1개월 내 운영자 전화/원격지원 건수 `50% 감소`
- [x] 도입 후 목표치는 월 `250건 이하`다.
- [x] 촬영 종료 후 추가 대기 시간 5분 이내 목표

### 5.3 정성 목표

- [x] 고객이 "이 매장은 내가 혼자 써도 된다"는 확신을 느껴야 한다.
- [x] 운영자는 "매장이 스스로 굴러간다"는 느낌을 받아야 한다.
- [x] 브랜드는 "무인인데도 부드럽다"는 인상을 줘야 한다.

## 6. 핵심 사용자

### 6.1 1차 사용자: 고객

고객은 촬영 경험의 주인공이다.

고객의 특징:

- 소프트웨어 숙련도가 높지 않을 수 있다.
- 매장 사용 시간이 제한되어 있다.
- 빨리 촬영하고, 잘 찍히고, 바로 다음 공간으로 이동하고 싶다.
- 실수에 대한 두려움이 있다.
- 기술 설명보다 "지금 무엇을 하면 되는지"를 원한다.

### 6.2 2차 사용자: 원격 운영자

운영자는 평소에는 개입하지 않다가 예외 상황에서만 개입한다.

운영자의 니즈:

- 현재 상태를 빠르게 파악하고 싶다.
- 필요한 경우 종료 시각 예외 연장을 하고 싶다.
- 전체 앱을 뒤엎지 않고 제한된 범위에서 복구하고 싶다.
- 고객에게 복잡한 설명을 하지 않고 빠르게 안정화하고 싶다.

### 6.3 3차 사용자: 오너/브랜드 운영자

직접 화면을 쓰는 비중은 낮지만, 제품의 실질 고객이다.

니즈:

- 여러 지점에 동일한 운영 품질 적용
- 교육비용 감소
- 문제 패턴 표준화
- 추후 예약/문자/후기/운영 자동화 확장 기반 확보

## 7. 고객 여정 맵

### 7.1 예약 전후

고객은 매장을 "전문 촬영 공간"이 아니라 "예약한 시간에 스스로 들어가 쓰는 무인 서비스"로 경험한다.  
따라서 방문 전부터 고객 머릿속에는 한 가지 질문이 있다.

`내가 혼자 가서 정말 할 수 있을까?`

이 불안을 줄이기 위해 제품은 방문 시점 이후에도 고객이 계속 확신을 얻도록 설계되어야 한다.

### 7.2 입장 직후

고객은 입장 후 가장 먼저 PC에서 예약자명과 휴대전화 뒤4자리를 입력해 자기 세션을 열어야 한다.

이 단계의 제품 역할:

- 고객이 봐야 할 화면을 하나로 고정
- 세션 체크인이라는 첫 행동을 명확히 제시
- 카메라 상태와 최소 준비 사항만 안내
- Windows나 다른 프로그램 노출 제거
- 가능하다면 고객이 `세팅 전 리모컨 조작 금지` 같은 내부 규칙을 배우지 않아도 되게 만들기

이 시점에 고객이 헷갈리면 곧바로 첫 전화가 발생한다.

고객이 이 순간 가장 먼저 헤매는 지점은 `다비 촬영 매니저` 같은 내부 앱명을 이해하는 것이 아니라 `지금 무엇부터 해야 하는가`다.  
따라서 첫 화면은 프로그램 이름 설명보다 `예약자명 + 휴대전화 뒤4자리 입력`이라는 첫 행동을 먼저 보여줘야 한다.

이 단계의 안내 문구 길이도 제한해야 한다.  
한 화면에는 핵심 안내 1문장과 보조 설명 1문장까지만 두고, 장문 절차 설명이나 복수의 선택지를 한 번에 읽게 해서는 안 된다.

시스템으로 사전 방지가 불가능한 경우에만 보조 경고문을 사용한다.  
이때의 최소 문구는 `화면에 촬영 가능이 표시될 때까지 리모컨을 누르지 마세요.`다.

### 7.3 세션 시작

고객은 예약자명과 휴대전화 뒤4자리를 입력해 자기 세션을 연다.
첫 체크인 화면의 기본 안내 문구는 `예약자명과 휴대전화 뒤4자리를 입력해 주세요.`다.
첫 체크인 화면의 보조 설명 문구는 `세션 확인용 정보입니다.`다.
단, 예약자명은 비워둘 수 없고 한글만 입력할 수 있어야 하며, 휴대전화 뒤4자리는 반드시 숫자 4자리 형식이어야 한다. 형식이 맞지 않으면 진행시켜서는 안 된다.
휴대전화 뒤4자리 입력칸에는 숫자 외 문자가 아예 입력되지 않도록 막아야 한다.
형식이 맞지 않을 때의 기준 문구는 `예약자명은 한글로 입력하고, 휴대전화 뒤4자리는 숫자 4자리로 입력해 주세요.`다.
이 형식 오류 문구는 `Enter`를 누르기 전에도 화면에서 빨간색으로 바로 보여야 한다.
체크인 화면이 열리면 커서는 처음부터 `예약자명` 입력칸에 자동으로 가 있어야 한다.
입력 형식이 맞으면 `Enter` 키만으로 바로 다음 단계로 넘어갈 수 있어야 한다.
체크인 화면에는 별도의 `다음` 버튼이 없어야 한다.
이 입력 경험은 키오스크용 화면 키보드가 아니라 데스크톱 PC와 모니터 환경을 전제로 한다.

이 단계의 목적은 고객을 통과시키거나 막기 위한 검증이 아니라 다음 세 가지다.

- 고객에게 "지금부터 내 촬영이 시작된다"는 심리적 시작점 제공
- 파일을 고객 단위로 분리 관리
- 이후 셀렉실에서 찾을 수 있는 이름 체계 확보

입력값이 실제 예약 정보와 정확히 일치하지 않아도 이 단계에서 고객 진행을 막지 않는다.
필요하면 운영에서는 시간대 기준으로 해당 세션을 사후 추정할 수 있어야 한다.
현장 기준으로 고객은 이 `예약자명 + 휴대전화 뒤4자리` 입력을 대체로 무리 없이 수행한다.
입력이 끝나면 고객은 별도 확인 단계 없이 바로 프리셋 선택으로 넘어간다.
세션명 형식은 `예약자명 + 휴대전화 뒤4자리`다.
같은 날 동일한 세션명이 다시 생기면 고객에게 보이는 세션명도 `_2`, `_3`처럼 뒤에 번호를 붙여 구분한다.
단, 전 타임 사진 혼선 방지를 위해 현재 고객 세션을 자동으로 새로 분리해야 하는 예외 상황에서는 `3월 2일 5시 예약손님` 같은 `월 일 시 예약손님` 형식의 자동 세션명을 쓸 수 있다.
이 예외 세션명은 고객에게 그대로 보여주되, `자동 복구`, `예외 처리` 같은 별도 설명 문구는 붙이지 않는다.

즉, 이 입력 과정은 운영을 위한 식별 입력이면서 동시에 고객 경험의 체크인 단계다.

### 7.4 프리셋 선택

프리셋은 단순한 옵션이 아니다.  
고객이 "오늘 어떤 무드로 찍을지"를 정하는 핵심 경험이다.

따라서 프리셋 선택은:

- 시작 전에 적용 프리셋이 정해져 있어야 하고
- 이름만이 아니라 예시 이미지로 직관적으로 보여야 하며
- 촬영 중에도 바꿀 수 있어야 하고
- 바뀐 프리셋이 언제부터 적용되는지 명확히 안내해야 한다

프리셋 이름은 추상적인 무드명이 아니라 고객이 바로 이해하는 설명형 이름을 쓴다.
기본 이름 체계 예시는 `웜톤`, `쿨톤`, `배경지 - 아이보리`, `배경지 - 핑크`다.
즉, 프리셋 분류 기준은 `색감/톤`과 `배경지`처럼 고객이 즉시 구분 가능한 기준이어야 한다.
프리셋 예시 이미지는 임의 샘플이 아니라 시스템에 현재 보이는 최신 실제 사진을 기준으로 써야 한다.
각 프리셋은 같은 최신 사진에 적용된 결과로 비교되어야 한다.
프리셋 선택 화면에서 고객에게 노출하는 프리셋 수는 최대 `6개`다.
기본 선택은 현재 세션과 무관하게, 전 타임이든 전전 타임이든 가장 마지막에 사용했던 프리셋으로 항상 미리 맞춰둔다.
고객은 이 미리 선택된 프리셋을 그대로 두고 별도 조작 없이 바로 촬영 가능 화면으로 넘어갈 수 있다.

MVP에서는 지점별로 다른 프리셋 경험을 허용하지 않는다.  
전 지점이 같은 프리셋 카탈로그, 같은 설명형 이름, 같은 등록순 노출 규칙을 공유한다.  
지점별로 다른 프리셋을 노출하기 시작하면 고객은 브랜드 안에서 다른 사용법을 다시 배워야 하고 운영 표준화도 깨진다.

프리셋 선택이 끝나면 고객은 별도 중간 단계 없이 바로 촬영 가능 화면으로 들어가야 한다.

이 경험이 어설프면 고객은 촬영보다 설정 확인에 시간을 쓴다.

### 7.5 촬영 중

촬영 중 고객이 원하는 것은 복잡한 정보가 아니다.

- 가능하면 시스템이 3초 간격의 촬영 신호를 대신 처리해 고객이 할 일을 없애는 것
- 그게 가능할 때는 `실제 촬영 종료 시각`만 크게 보여주는 것
- 그게 불가능할 때만 `리모컨 3초에 한 번`과 `실제 촬영 종료 시각`을 함께 보여주는 것

남은 시간을 함께 보여줄 경우, 이는 보조 정보여야 하며 `초 단위`까지 카운트다운되어야 한다.
현재 적용 중인 프리셋 이름도 촬영 가능 화면에서 계속 보여줘야 하지만, 종료 시각보다 크게 강조할 필요는 없다.

즉, 이 화면의 목표는 정보를 많이 주는 것이 아니라 고객이 지금 무엇을 해야 하는지 헷갈리지 않게 만드는 것이다.

여기서 중요한 전제는 고객이 `라이트룸이 어떤 상태인지`, `팝업이 떠 있는지`, `어느 화면에서 종료했는지`를 이해할 필요가 없어야 한다는 점이다.  
좋은 제품은 고객이 내부 프로그램 상태를 관리하게 하지 않고, 시스템이 `지금 촬영 가능한가 아닌가`를 먼저 분명하게 판단해주는 쪽이어야 한다.
즉, 제품 원칙은 `경고문으로 고객을 교육`하는 것이 아니라 `시스템이 먼저 감지하고 막는 것`이다.  
경고문은 그 방지가 불가능한 경우에만 차선책으로 사용한다.

특히 `1시간 추가 쿠폰` 고객은 세션 시작 직후부터 기본 고객과 다른 시간 체계를 보아야 한다.  
연장 고객이 처음에는 기본 종료 시각으로 보다가 나중에 시간이 늘어나는 방식이면, 시스템이 불안정하거나 쿠폰이 반영되지 않았다고 해석하기 쉽다.  
즉, 연장 고객에게는 시작 시점부터 `연장된 촬영 종료 시각`이 바로 보여야 한다.  
이때 고객 화면에는 `전체 이용 종료 시각`까지 함께 보여주지 않고, 지금 고객 행동과 직접 연결되는 `촬영 종료 시각`만 크게 보여준다.  
또한 `쿠폰 적용됨` 같은 별도 문구를 추가로 설명하지 않고, 시간 정보 자체가 이미 연장 상태로 보이게 한다.  
운영자 예외 연장이 적용되는 경우에도 고객 화면의 `촬영 종료 시각`은 즉시 새 시각으로 갱신되어야 한다.
이 경우에는 시작 시점 쿠폰 고객과 달리, 고객에게 `시간이 연장되었다`는 사실을 한 번 명확히 알려주는 별도 문구를 노출한다.
즉, 쿠폰 종료 알림 혼선은 별도 추적 지표보다 `촬영 시작 전 실제 종료 시각 명시`로 해결하는 문제로 본다.

특히 최근 사진이 크게 보여야 하는 이유는 고객이 그 한 장만 보고도 아래를 판단하기 때문이다.

- 구도
- 표정
- 프리셋 느낌
- 촬영이 정상 저장되고 있는지

즉, 최근 사진 미리보기는 편의 기능이 아니라 `불안 제거 기능`이다.

고객에게 추가로 보여줄 정보도 제한한다.  
촬영 중 고객이 알아야 하는 것은 `지금 촬영 가능한가`, `언제 끝나는가`, `방금 찍힌 사진이 잘 저장되었는가` 세 가지다.  
따라서 저장 경로, 내부 프로그램 상태, 운영자용 진단 정보는 고객 화면에 올리지 않는다.

최근 사진 크게 보기 외의 보조 안심 요소는 `자기 세션 사진만 보이는 것`, `썸네일이 계속 쌓이는 것`, `실제 종료 시각이 안정적으로 보이는 것` 정도면 충분하다.

### 7.6 촬영 종료 직전

종료 직전 경고와 종료 알림은 단순 알림이 아니라 `무인 운영에서 고객 동선을 끊김 없이 이동시키는 장치`다.

고객은 시간이 다 되어갈수록 불안해진다.

- 아직 사진이 다 처리됐는지
- 지금 더 찍어도 되는지
- 끝나면 무엇을 해야 하는지

따라서 종료 직전 UX는 반드시 예측 가능해야 한다.

- 실제 종료 시각 5분 전에는 미리 정리할 시간을 준다.
- 실제 종료 시각에는 촬영을 명확히 종료한다.
- 종료 시각이 되면 고객 화면은 즉시 촬영 불가 상태로 바뀌어야 한다.
- 내보내기가 아직 100% 끝나지 않았다면 `내보내기 중` 상태를 먼저 분명히 보여줘야 한다.
- `내보내기 중` 상태에서는 고객이 자리를 떠나지 않고 그 자리에서 완료까지 기다리게 해야 한다.
- `내보내기 중` 상태에는 현재 진행률이 보여야 한다.
- 내보내기가 완료되는 즉시 다음 공간으로 이동할 이름을 정확히 보여준다.

특히 `1시간 추가 쿠폰` 고객에게는 기본 세션 기준의 `n시 50분 종료` 경험이 그대로 보이면 안 된다.  
고객은 이를 "시간이 끝났다", "쿠폰이 반영되지 않았다", "시스템이 오류다"로 해석하기 쉽다.  
연장 세션의 종료 알림 기준 시각은 `예약 시작 시각 + 1시간 40분`이어야 한다. 예를 들어 `10:00` 시작 고객이면 종료 알림은 `11:40`에 떠야 한다.  
즉, 종료 직전 UX는 일반 세션과 연장 세션을 고객이 혼동하지 않도록 구분되어야 한다.

또한 연장 고객은 종료 직전이 되어서야 예외 처리를 인지하면 안 된다.  
세션 시작 직후부터 종료 기준이 이미 연장 상태로 보여야 한다.

종료 5분 전 경고의 기준 문구:

`촬영 종료까지 5분 남았습니다. 필요한 사진을 마무리해 주세요.`

실제 종료 시각의 기준 문구:

`촬영이 종료되었습니다. 내보내기를 시작합니다. 잠시만 기다려 주세요.`

운영자 예외 연장 적용 시 기준 문구:

`촬영 시간이 연장되었습니다. 변경된 종료 시각을 확인해 주세요.`

이 두 문구는 일반 고객과 연장 고객 모두 동일하게 쓰되, 노출 시각만 각 고객의 실제 종료 시각 기준으로 달라진다.
운영자 예외 연장이 적용되면 종료 5분 전 경고와 종료 2분 전 전화 전환 기준도 새 종료 시각 기준으로 다시 계산한다.

종료 후 대기 경험은 "멈춤"처럼 보이면 안 된다.  
고객은 지금 시스템이 내 사진을 준비하고 있다는 느낌을 받아야 한다.  
따라서 `내보내는 중입니다. [몇 %]`처럼 현재 작업과 진행 정도를 먼저 보여주고, 진행률이 잠시 멈출 때도 `시간이 조금 더 걸리고 있습니다`처럼 작업이 이어지고 있음을 설명해야 한다.  
완료 순간에는 기다림의 이유를 더 설명하지 말고 `세션명`과 다음 행동을 즉시 보여줘야 한다.

### 7.7 결과 확인 단계

촬영실 경험의 마지막 목표는 "끝났다"가 아니다.  
`다음 단계로 정확히 인계되었다`가 목표다.

현재 MVP에서 그 인계 지점은 별도 앱이 아니라 바탕화면의 `사진저장폴더`다.  
추후에는 이 단계를 별도 결과 확인 앱으로 대체할 수 있다.

완료 화면은 고객에게 아래 두 가지를 동시에 전달해야 한다.

- 사진 준비가 끝났다는 안심
- 바탕화면의 `사진저장폴더`에서 찾아야 할 `세션명` 하나에 대한 명확한 지시

즉, 촬영실 제품의 마지막 화면은 종료 화면이 아니라 `인계 화면`이다.
단, 내보내기가 아직 끝나지 않았다면 완료 안내보다 먼저 `내보내기 중` 상태를 보여줘야 한다.
이때 고객은 바로 이동하는 것이 아니라, 현재 자리에서 완료 안내가 뜰 때까지 기다려야 한다.
또한 이 대기 화면에는 현재 진행률이 보여야 한다.
운영자 예외 연장이 `내보내기 중`에 적용되면 현재 내보내기를 끝낸 뒤, 연장된 종료 시각으로 다시 촬영 가능 화면으로 돌아가야 한다.
이때 추가 촬영 결과물은 새 세션을 만들지 않고 기존 세션 폴더에 계속 쌓여야 한다.
고객 유형과 무관하게, 각 고객의 실제 종료시간 2분 전까지 내보내기가 완료되지 않으면 전화 안내로 전환해야 한다.

`내보내기 중` 상태의 기준 문구:

`내보내는 중입니다. [몇 %]`

진행률이 한동안 변하지 않을 때의 기준 문구:

`내보내는 중입니다. 시간이 조금 더 걸리고 있습니다. 잠시만 기다려 주세요. [몇 %]`

종료시간 2분 전까지 내보내기가 완료되지 않았을 때의 기준 문구:

`내보내기가 지연되고 있습니다. 전화해 주세요.`

이 화면에는 고객이 바로 볼 수 있는 전화번호가 함께 표시되어야 한다.
전화번호는 지점별로 다를 수 있으며, 지점별 번호가 설정되지 않았을 때의 기본값은 `010-0000-0000`이다.

완료 화면의 기준 문구:

`내보내기가 완료됐습니다. 셀렉실로 이동하여 '바탕화면'의 '사진저장폴더'에서 [세션명]을 열어 셀렉 및 인화를 시작하세요.`

## 8. 운영 여정

### 8.1 평상시 운영

운영자의 이상적인 상태는 "아무것도 하지 않는 것"이다.

좋은 무인 운영 제품은 운영자에게 많은 기능을 주는 제품이 아니라, 운영자가 개입할 이유를 줄이는 제품이다.

### 8.2 예외 상황 운영

운영자가 개입하는 순간은 제한적이어야 한다.

- 고객 진행 화면에서 시간 예외를 허용해야 할 때
- 세팅 화면에서 현재 상태를 빠르게 파악해야 할 때
- 세팅 화면에서 필요한 설정을 조정해야 할 때

운영자 예외 연장은 촬영 중과 `내보내기 중` 모두 허용하며, `1시간` 단위로 적용한다. 각 연장은 버튼을 누른 시각이 아니라 그 시점의 `기존 종료 시각` 기준으로 더한다. 필요하면 상한 없이 반복 적용할 수 있다. 예외 연장이 적용되면 고객 화면의 종료 시각도 즉시 갱신되어야 한다.

운영자는 고객보다 더 많은 정보를 보되, 운영자 모드 역시 "딱 필요한 만큼만" 있어야 한다.  
운영자 화면이 과도하게 복잡하면 무인 운영이 아니라 원격 운영 소프트웨어가 된다.

운영자가 실제로 자주 수행하는 예외 조치는 다섯 가지로 정리된다.  
`매크로 오류 시 라이트룸 클래식을 직접 열어 촬영 세팅 수동 처리`, `매크로 오류 시 라이트룸 클래식에서 내보내기 직접 수행`, `카메라/앱 재시작 또는 촬영 시작 흐름 재시도`, `종료 시각 예외 연장`, `전 타임 사진 혼선 시 세션 분리`다.  
이 제품은 운영자에게 많은 조작 권한을 주는 대신, 이 다섯 가지 예외를 빠르게 처리할 수 있는 상태 정보를 우선 제공해야 한다.

특히 내보내기 문의는 대부분 촬영 종료 직후 `:50~정시` 사이의 짧은 내보내기 구간에서 발생한다.  
고객이 몇 분간 직접 시도하다가 실패한 뒤 전화하는 경우가 많기 때문에, 운영자는 다음 타임 촬영과 겹치지 않도록 수동 내보내기 노하우로 빠르게 마무리할 수 있어야 한다.

운영자에게 먼저 필요한 정보는 `현재 세션명`, `현재 단계`, `실제 종료 시각`, `연장 고객 여부`, `카메라 준비 상태`, `저장 경로 상태`, `최근 오류 5건`이다.  
그 이상의 상세 기술 로그는 MVP 고객/운영 경험 요구사항으로 보지 않는다.

운영자 권한 범위는 제품 내부 기준으로 제한된다.  
고객 진행 화면에서는 시간 연장만 허용하고, 운영자 세팅 화면에서는 상태 확인, 재시작, 내보내기 재시도, 세션 분리 복구를 허용한다.  
가격 약속, 환불 승인, 예약 정책 예외 같은 상업적 결정은 제품 밖 운영 권한이다.

- [x] 경고문은 시스템으로 방지할 수 없는 경우에만 보조 수단으로 사용한다.

- [x] 최소 운영 로그는 KPI 계산과 운영자 복구 판단에 필요한 세션 라이프사이클을 남겨야 한다.

## 12. 자동화 요구사항

이 제품의 핵심 경쟁력은 "자동화"다.  
자동화는 단순 편의가 아니라 무인 매장 운영을 성립시키는 운영 장치다.

이번 MVP에서 자동화되어야 하는 것:

- [x] 세션 이름 생성
- [x] 고객 폴더 생성
- [x] RAW 처리 후 결과물 준비
- [x] 고객 유형별 실제 종료 시각 기준 시간 경고 및 종료 알림
- [x] 종료 후 남은 작업 마무리
- [x] 완료 후 다음 정시 초기화

이번 MVP에서 가치가 가장 큰 자동화 우선순위는 아래 순서다.

1. `촬영 시작 준비 상태 자동 판정과 시작 진입 자동화`
2. `고객 유형별 실제 종료 시각 계산과 경고/종료 알림 자동화`
3. `내보내기 결과물 준비와 세션 인계 자동화`

향후 확장 대상으로 보는 자동화:

- [ ] 예약 검증
- [ ] 구글시트 기반 고객 데이터 대조
  - 첫 번째 활용 목적: `1시간 추가 쿠폰 고객의 연장 종료 시각 반영`
- [ ] 지점별 운영 설정 중앙관리
- [ ] 사전 안내 메시지/후기 요청 등 운영 자동화

예약/구글시트 연동이 먼저 해결해야 하는 운영 문제는 `연장 고객의 종료 시각 오표시`와 `다음 타임 충돌 확인`이다.  
즉, 예약 데이터 연동의 첫 가치는 예약 검증 자체보다 `고객 시간 경험을 틀리지 않게 만드는 것`이다.

- [x] 종료 5분 전 기준 문구는 `촬영 종료까지 5분 남았습니다. 필요한 사진을 마무리해 주세요.`다.
- [x] 실제 종료 시각 기준 문구는 `촬영이 종료되었습니다. 내보내기를 시작합니다. 잠시만 기다려 주세요.`다.
- [x] 운영자 예외 연장이 적용되면 종료 5분 전 경고와 종료 2분 전 전화 전환 기준은 새 종료 시각 기준으로 다시 계산해야 한다.
- [x] 완료 화면은 현재 MVP 기준 `사진저장폴더` 인계를 위한 화면이어야 한다.
- [x] 종료 시각이 되면 고객 화면은 즉시 촬영 불가 상태로 바뀌어야 한다.
- [x] 내보내기가 끝나지 않았다면 `내보내기 중` 상태를 먼저 보여줘야 한다.
- [x] `내보내기 중` 상태에서는 고객이 자리를 떠나지 않고 완료까지 기다려야 한다.
- [x] `내보내기 중` 상태에는 진행률이 보여야 한다.
- [x] 고객 유형과 무관하게 실제 종료시간 2분 전까지 내보내기가 완료되지 않으면 전화 안내를 보여줘야 한다.
- [x] 전화 안내 화면에는 즉시 확인 가능한 전화번호가 함께 보여야 한다.
- [x] 전화번호는 지점별 값으로 관리하고, 기본값은 `010-0000-0000`이다.
- [x] 지점별로 고객에게 달라질 수 있는 제품 설정은 현재 전화번호 하나만 허용한다.
- [x] `촬영 준비 중입니다. 잠시만 기다려 주세요` 상태도 실제 종료시간 2분 전까지 지속되면 전화 안내를 보여줘야 한다.
- [x] 촬영 직후 현재 고객 세션 폴더에 새 사진이 생성되지 않거나 더 이상 쌓이지 않으면 즉시 전화 안내를 보여줘야 한다.
- [x] 위 상황의 기준 문구는 `촬영한 사진이 저장되지 않고 있습니다. 전화해 주세요.`다.
- [x] 전 타임 사진 혼선이 정확히 감지되고 현재 고객 세션 분리가 가능하면 전화보다 자동 복구를 우선해야 한다.
- [x] 내보내기 완료 후 앱이 `사진저장폴더`를 자동으로 열 필요는 없다.
- [x] `사진저장폴더` 이동 안내 화면에서 고객이 반드시 기억해야 할 정보는 `세션명` 하나다.
- [x] 고객 진행 화면의 운영자 기능은 예외 연장만 허용한다.
- [x] 운영자 예외 연장은 촬영 중과 `내보내기 중` 모두 허용하며, 1시간 단위이고 `기존 종료 시각` 기준으로 상한 없이 반복 적용 가능하다.
- [x] 예외 연장이 적용되면 고객 화면의 실제 촬영 종료 시각도 즉시 갱신되어야 한다.
- [x] 예외 연장이 적용되면 고객에게 시간 연장 안내 문구도 함께 보여줘야 한다.
- [x] `내보내기 중` 예외 연장 후 추가 촬영 결과물은 기존 세션 폴더에 계속 쌓여야 한다.
- [x] `내보내기 중` 예외 연장이 적용되면 현재 내보내기를 끝낸 뒤 촬영 가능 화면으로 돌아가야 한다.
- [x] 별도의 운영자 세팅 화면에는 각종 설정과 상태 확인 기능이 필요하다.
- [x] 운영자 세팅 화면의 첫 상태는 `현재 촬영 가능 여부`여야 한다.
- [x] 운영자 화면의 첫 상태는 `촬영 가능 / 준비 중 / 전화 필요` 3단계면 충분하다.
- [x] 운영자 화면의 상태 3단계는 색상으로도 구분되어야 한다.
- [x] 운영자 상태 색상은 `초록=촬영 가능`, `노랑=준비 중`, `빨강=전화 필요`다.
- [x] `전화 필요` 상태에는 운영자가 바로 이해할 수 있는 원인이 함께 보여야 한다.
- [x] `전화 필요` 상태에는 최근 원인 5개가 함께 보여야 한다.
- [x] 연장 고객에게는 별도 쿠폰 안내 문구 없이 연장된 종료 시각만 보여준다.
- [x] 촬영 준비 중 상태에서는 고객 조작을 완전히 막는다.
- [x] 고객 실수 방지는 시스템 감지/방지를 우선하고, 불가능할 때만 경고문을 쓴다.
- [x] 시스템 방지가 불가할 때의 최소 경고문은 `화면에 촬영 가능이 표시될 때까지 리모컨을 누르지 마세요.`다.
- [x] 최소 운영 로그는 세션 라이프사이클과 운영자 개입 유형을 KPI 계산 가능 수준으로 남겨야 한다.
- [x] 운영 로그는 민감정보를 새로 저장하지 않고도 `촬영 시작/내보내기/쿠폰 혼선/기타` 분류가 가능해야 한다.

## 20. 결론

Boothy의 이번 MVP는 "라이트룸을 대체하는 앱"이 아니다.  
정확히는 `무인 셀프사진관의 촬영 시작 실패와 종료 혼선을 줄이기 위한 고객 여정 자동화 제품`이다.

이 제품이 잘 만들어졌다는 뜻은 기능이 많다는 뜻이 아니다.  
고객이 혼자 와서, 혼자 시작하고, 혼자 끝내고, 운영자는 개입하지 않아도 되는 상태가 되었다는 뜻이다.
