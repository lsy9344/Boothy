# Story 1.9: Capture Trigger, Ingestion, and Auto-Apply Capture Preset

## Status
Split (see Stories 1.9.1–1.9.3)

## Story
**As a** studio customer,  
**I want** to press one big “Shoot” button and immediately see the photo with my selected filter applied,  
**so that** I get instant feedback without manual importing or editing.

Scope: This is an umbrella story for real tether capture for EOS 700D via Canon EDSDK on Windows: (1) camera session connect/disconnect/reconnect + basic health monitoring + event-driven status updates, (2) shutter trigger from Customer Mode, (3) file ingestion/download into the active session folder, and (4) auto-create sidecar from the current capture preset for **newly captured images only**.

This story was too large to implement safely as a single unit, so it is split into:
- Story 1.9.1: `docs/stories/1.9.1.tethering-controller-session-lifecycle-status-events.md`
- Story 1.9.4: `docs/stories/1.9.4.wm-devicechange-physical-connection-detection.md`
- Story 1.9.2: `docs/stories/1.9.2.shutter-trigger-and-ingestion-to-session-folder.md`
- Story 1.9.3: `docs/stories/1.9.3.auto-apply-capture-preset-sidecar-auto-create.md`

Out of scope: Live View embedding (Story 1.10) and camera-setting lock/guardrails (FR16 / Story 1.12+).

## Acceptance Criteria
1. Customer Mode의 촬영 버튼이 EDSDK 셔터 트리거를 호출한다(우선 경로).
2. 촬영 파일은 `PC(Host)`의 활성 세션 폴더로 저장되며, 수신 완료 후 UI에 자동 로드되고 최근 촬영본이 자동 선택된다.
3. sidecar가 없으면 “현재 캡처 프리셋”으로 초기값을 생성하고 sidecar를 생성한다.
4. 프리셋 변경 이후 촬영분부터 새 프리셋이 적용되며, 과거 촬영분은 자동 변경되지 않는다.
5. 이벤트 중복/누락/부분 전송 등 엣지 케이스에서 중복 처리 없이 안정적으로 동작한다(아이템포턴시).
6. EDSDK 기반 카메라 세션(연결/해제/재연결) + 기본 헬스 모니터링(백그라운드) 및 이벤트 기반 상태 업데이트를 구현한다. (Story 1.8에서 만든 상태 모델/진단 UI/게이팅 스캐폴딩을 “실제 상태”로 구동)
   - Admin Diagnostics의 Connect/Disconnect/Retry 액션을 실제 동작으로 연결한다.
   - Customer/Admin UI는 상태 변경을 자동 반영한다(수동 Refresh 의존 최소화).

## Dev Notes
- Customer Mode must not expose filesystem paths or diagnostics (Admin-only).
- Destination folder for captures is always the active session folder.
- Sidecar auto-create applies to newly captured images only; never rewrite existing sidecars.
