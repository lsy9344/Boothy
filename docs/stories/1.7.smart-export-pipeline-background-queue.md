# Story 1.7: Smart Export Pipeline (Background Queue)

## Status
Ready for Review

(QA fixes applied 2026-01-04 - Third round: OPS-001 canonicalize fallback + improved test coverage)

## Story
**As a** studio customer,  
**I want** my photos to be processed in the background during shooting,  
**so that** waiting time at the end of the session is minimized.

## Acceptance Criteria
1. 새 이미지가 세션 폴더에 추가되면 백그라운드 큐에 JPEG 생성 작업이 enqueue된다.
2. Customer Mode에는 “처리 중(남은 n장)” 수준의 큐 상태를 표시한다.
3. Admin Mode에는 실패 항목/재시도/세부 상태를 표시한다.
4. 큐가 프리뷰/라이브뷰를 방해하지 않도록 우선순위/스로틀링 정책을 적용한다.
5. 세션 종료(ExportLock) 시 큐를 drain하여 남은 항목을 처리하고 완료 후 다음 단계로 진행한다.
6. Smart Export를 끄면 “종료 시 일괄 Export”로 롤백 가능하다.

## Dev Notes

### Goal / Context
- This story introduces a background queue to process session photos during the session, reducing end-of-session waiting and aligning with kiosk non-freezing UX requirements.
  [Source: docs/prd/epic-1-rapidtetherraw-customer-mode-mvp.md#story-17-smart-export-pipeline-background-queue; docs/prd/requirements.md#non-functional (NFR2)]

### Previous Story Insights
- A session-scoped export pipeline already exists (Story 1.6) including progress events and explicit cancellation semantics; Smart Export should reuse the same pipeline rather than creating a parallel JPEG export implementation.
  [Source: docs/stories/1.6.export-rules-filename-templating.md#dev-agent-record]
