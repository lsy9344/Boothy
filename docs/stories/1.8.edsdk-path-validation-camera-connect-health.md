# Story 1.8: EDSDK Path Validation & Camera Connect/Health

## Status
Done

## Story
**As a** studio operator,  
**I want** the kiosk to validate EDSDK and maintain stable camera connectivity,  
**so that** the session cannot start in a broken hardware/software state.

Scope: (1) Admin EDSDK path validation + diagnostics, (2) camera status model/diagnostics UI surfaced to Customer/Admin UI, (3) Customer Mode gating so “Start Capturing / Shoot” is blocked when tethering is required but not ready. This story does **not** implement the actual shutter trigger / ingestion pipeline (Story 1.9).
Scope note: 본 스토리는 **카메라 상태 모델/진단 UI/게이팅 스캐폴딩**까지 포함한다. 실제 EOS 700D 연결/해제/재연결 및 헬스 모니터링(백그라운드)과 이벤트 기반 상태 업데이트는 Story 1.9에서 구현한다.

## Acceptance Criteria
1. Admin에서 EDSDK 경로를 설정하고, 필수 구성요소(DLL/런타임) 존재 여부를 검증한다(Windows).
2. EOS 700D 테더링을 위한 **카메라 상태 모델/진단 UI/게이팅 스캐폴딩**을 제공하고, 상태를 UI에 표시한다(Customer는 단순, Admin은 상세). (실제 연결/해제/재연결/헬스 모니터링은 Story 1.9)
3. 카메라 상태가 not-ready/error일 때 안전 상태로 전환하고 재시도/도움 요청 경로를 제공한다. (실제 케이블 분리/절전 감지 및 자동 재연결은 Story 1.9)
4. Customer Mode는 카메라가 준비되지 않으면 Capture로 진행할 수 없다(자동 점검 + 차단).
5. 테더 기능을 비활성화하면 파일 기반(수동) 모드로 롤백 가능하다(운영 플랜 B).

## Dev Notes

### Goal / Context
- EDSDK는 Windows-only이며(비동봉/비재배포), 키오스크 운영에서 “경로 설정 실패/카메라 미연결”은 곧바로 세션 진행 불가로 이어지므로, Idle/Setup 단계에서 자동 점검 + 차단 + Admin 진단이 필요하다.
  [Source: docs/architecture/tech-stack.md#external--constraints; docs/prd/requirements.md#functional (FR24); docs/prd/technical-constraints-and-integration-requirements.md#deployment-and-operations]

### Previous Story Insights
- `edsdkPath`는 이미 KioskConfig에 존재하며, 현재는 “존재 여부” 수준의 경고만 제공한다. Story 1.8에서는 이를 “필수 구성요소 검증 + 상세 진단(Windows)”으로 확장한다.
  [Source: upstream/RapidRAW/src/types/kioskConfig.ts; upstream/RapidRAW/src-tauri/src/kiosk_config.rs]
- Customer Mode UI는 “짧고 구체적 + 도움 요청 1가지 액션”만 제공하고, 상세 원인/로그/재시도는 Admin 전용으로 둔다.
  [Source: docs/architecture/coding-standards.md#frontend-reacttypescript; docs/prd/user-interface-enhancement-goals.md#ui-consistency-requirements]
- Windows-only 기능은 플랫폼 가드/feature flag로 격리해 macOS 개발 흐름을 막지 않는 것이 표준이다.
  [Source: docs/architecture/coding-standards.md#backend-rusttauri]

### Tech Stack (Relevant)
- Tauri 2 + Rust 백엔드에서 EDSDK(Windows-only, 로컬 설치)를 경로 설정으로 사용한다. EDSDK 파일은 repo/installer에 포함하지 않는다.
  [Source: docs/architecture/tech-stack.md#external--constraints; docs/prd/requirements.md#non-functional (NFR8)]
