## Critical Risks Requiring Immediate Attention

### 1) REL-002: ExportLock drain can return early

**Score: 9 (Critical)**  
**Probability**: High - background worker is not stopped/awaited during drain, and drain does not re-wait for in-progress work after the queue becomes empty.  
**Impact**: High - ExportLock can proceed before Smart Export completes, violating AC5 and risking incorrect “done” UI/flow.

**Mitigation (preventive)**:
- Stop/pause background worker during drain and ensure drain blocks until all jobs are terminal.
- Add a deterministic “draining” state so background worker does not pop jobs while drain is active.

**Testing Focus**:
- Integration: background worker running + ExportLock drain must not return until worker finishes last job.
- UI: ExportLock must not display “done” while Smart Export is still in-progress.

## Risk-Based Testing Strategy

### Priority 1 (Critical)

- Smart Export enabled → enqueue occurs → background worker processes jobs.
- ExportLock drain blocks until completion even if worker is running.

### Priority 2 (High)

- Load test with many images: UI remains responsive; verify worker throttling is sufficient.

### Priority 3 (Medium/Low)

- Handle “file not finalized yet” cases (canonicalize failures) without silently missing jobs.
