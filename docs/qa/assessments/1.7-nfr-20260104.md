# NFR Assessment: 1.7

Date: 2026-01-04
Reviewer: Quinn (Test Architect)

## Summary

- Security: PASS - Smart Export session-boundary validation now canonicalizes paths before `starts_with()`, reducing traversal/symlink bypass risk.
- Performance: CONCERNS - Background processing exists and is throttled (100ms between jobs), but there’s no evidence-based validation that preview/live view remains unaffected under load.
- Reliability: FAIL - `smart_export_drain` can return while the background worker is still processing a job (worker is not stopped or awaited), so ExportLock may proceed before Smart Export completes.
- Maintainability: CONCERNS - Unit tests exist for queue basics, but missing coverage for drain/worker interaction and end-to-end behavior.

## Evidence Reviewed

- Frontend: `npm run build`, `npm test -- --run`
- Backend: `cargo check`, `cargo test`

## Critical Issues

1. **ExportLock drain is not a true drain when worker is running** (Reliability)
   - Risk: Background worker can pop and process a job after `smart_export_drain` begins; `drain()` breaks once the queue is empty and does not re-wait for `is_processing`, so it can return early while a job is still in-progress.
   - Fix: Stop/pause the background worker during drain and/or wait for all in-progress work to finish after the queue is empty.

## Quick Wins

- Add tests for drain/worker interaction: worker running + ExportLock drain must block until all jobs are terminal (completed/failed).
