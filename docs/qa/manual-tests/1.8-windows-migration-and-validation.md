이 문서는 **macOS에서 작업 중인 Boothy 프로젝트를 Windows PC로 옮긴 뒤**, Story 1.8(EDSDK 경로 검증 + Customer 게이팅/롤백)을 **초보자도 그대로 따라 할 수 있게** 단계별로 안내합니다.

## 3) Windows 개발 환경 설치 (처음 1회)

Tauri 앱 실행에 필요합니다.
- 대부분 Windows 11은 기본 포함

## 5) Windows에서 EDSDK 파일 배치(어디에 두어야 하나?)

Story 1.8의 검증 로직은 **“EDSDK 루트 폴더” 아래에 `Dll/`과 `Header/`가 있는 구조**를 기대합니다.

검증에서 **필수로 보는 파일(중요)**:
- `Dll\EDSDK.dll`
- `Dll\EdsImage.dll`

검증에서 **참고용(없어도 되지만 경고로 표시)**:
- `Header\EDSDK.h`
- `Header\EDSDKTypes.h`
- `Header\EDSDKErrors.h`

### 5-1) 권장 배치 위치(초보자용, 실수 방지)
EDSDK는 프로젝트 소스와 분리된 고정 경로를 추천합니다(실수로 Git에 포함되는 것 방지).
- 예: `C:\BoothySDK\Canon\EDSDK\`

최종 구조 예시:
```
C:\BoothySDK\Canon\EDSDK\
  Dll\
    EDSDK.dll
    EdsImage.dll
  Header\
    EDSDK.h
    EDSDKTypes.h
    EDSDKErrors.h
  Document\   (선택: PDF/ReleaseNote 등 참고 문서)
```

### 5-2) macOS에 있는 SDK에서 복사하는 방법(권장)
macOS 프로젝트에 `SDK_for_developers/`가 있다면 다음을 Windows로 복사하세요.

1) 64-bit DLL 복사(권장):
- 원본: `SDK_for_developers/Windows/EDSDK_64/Dll/`
- 대상: `C:\BoothySDK\Canon\EDSDK\Dll\`

2) Header 복사(참고용 경고 제거):
- 원본: `SDK_for_developers/Windows/EDSDK/Header/`
- 대상: `C:\BoothySDK\Canon\EDSDK\Header\`

3) 참고 문서(PDF 등) 복사(선택):
- 원본: `SDK_for_developers/Document/`
- 대상: `C:\BoothySDK\Canon\EDSDK\Document\`

> 참고: `SDK_for_developers/Windows/EDSDK_64/`에는 Header 폴더가 없을 수 있어, Header는 `EDSDK/Header`에서 가져오는 구성이 가장 무난합니다.

## 6) 앱에서 EDSDK 경로 설정(가장 중요)

### 6-1) Admin Mode로 들어가기
- 앱이 “Customer Mode Unavailable”로 막혀 있으면:
  - PIN 입력 칸에 아무 값(예: `0000`)을 넣고 `Enter Admin Mode` 클릭  
  - (PIN이 설정되어 있지 않으면 어떤 값도 통과하도록 구현되어 있습니다)
- Customer Mode 화면이라면:
  - `Admin Access` 버튼 → PIN 입력 → Admin으로 전환

### 6-2) Admin Settings 열기
Admin 화면에서 **좌측 하단 톱니바퀴(Boothy Settings)** 버튼 클릭

### 6-3) Kiosk Configuration에서 EDSDK 경로 지정
1. 탭: `Kiosk Configuration`
2. 섹션: `Canon EDSDK Path (Windows-only)`
3. `Browse` 클릭
4. 아래 파일을 선택:
   - `C:\BoothySDK\Canon\EDSDK\Dll\EDSDK.dll`
5. `Save Configuration` 클릭

> 앱은 파일 경로(EDSDK.dll)를 받아도 내부에서 “EDSDK 루트”로 정규화하여 `Dll/`, `Header/`를 검사합니다.

## 7) Story 1.8 Windows 수동 검증 체크리스트(증거 남기기용)

아래 시나리오를 수행하면서 **스크린샷을 남기면** gate CONCERNS 해소에 도움이 됩니다.

### 공통: Camera Diagnostics 화면 열기
1. Admin Settings(톱니바퀴) → 탭 `Camera Diagnostics`
2. `Run Full Diagnostics` 또는 `Re-run Check` 클릭

---

### 시나리오 A) EDSDK 미설정 상태(테더링 OFF 기본 동작)
**목표:** edsdkPath가 비어 있으면 테더링 기능이 꺼지고, 수동(파일 기반) 모드로 안내되는지 확인

Steps:
1. Admin Settings → `Kiosk Configuration` → EDSDK Path 입력칸을 비움 → `Save Configuration`
2. Admin Settings → `Camera Diagnostics` 탭

Expected:
- “EDSDK path is not configured…” 안내가 보임
- “Without tethering, the system will use file-based (manual) mode…” 문구가 보임

Pass:
- [ ] 안내 문구가 예상대로 표시됨

증거(권장):
- [ ] `Camera Diagnostics` 화면 스크린샷 1장

---

### 시나리오 B) EDSDK 잘못된 경로(검증 실패)
**목표:** 잘못된 경로를 넣으면 “Invalid EDSDK path”로 실패하고 missing DLL이 표시되는지 확인

Steps:
1. Admin Settings → `Kiosk Configuration`
2. EDSDK Path 입력칸에 존재하지 않는 경로를 직접 입력(예: `C:\BoothySDK\Canon\EDSDK\Dll\NOPE.dll`)
3. `Save Configuration`
4. `Camera Diagnostics` 탭 → `Re-run Check`

Expected:
- 빨간 요약 박스(실패)
- “Invalid EDSDK path …” 또는 “EDSDK path does not exist” 계열 문구
- Missing Required DLLs에 `EDSDK.dll`, `EdsImage.dll` 표시(또는 경로 오류로 전체 누락 표시)

Pass:
- [ ] 실패 요약/누락 항목이 표시됨

증거(권장):
- [ ] `Camera Diagnostics`(실패) 스크린샷 1장

---

### 시나리오 C) 올바른 EDSDK 경로(검증 성공 + 정규화 확인)
**목표:** `Dll\EDSDK.dll`을 선택해도 **루트가 올바르게 잡히고**(정규화), required DLL이 모두 있다고 표시되는지 확인

Steps:
1. Admin Settings → `Kiosk Configuration` → `Browse`
2. `C:\BoothySDK\Canon\EDSDK\Dll\EDSDK.dll` 선택
3. `Save Configuration`
4. `Camera Diagnostics` 탭 → `Re-run Check`

Expected:
- 초록 요약 박스(성공): “EDSDK installation is valid and ready for use”
- EDSDK Root가 `...\EDSDK\`(루트)로 표시( `...\EDSDK\Dll` 로 표시되면 정규화 실패 가능성)
- Missing Required DLLs가 비어 있음
- Header를 복사해뒀다면 Missing Header Files도 비어 있음(없어도 “non-critical”로만 표시)

Pass:
- [ ] 성공 요약이 표시됨
- [ ] Root가 루트 폴더로 표시됨
- [ ] Missing Required DLLs 없음

증거(권장):
- [ ] `Camera Diagnostics`(성공) 스크린샷 1장

---

### 시나리오 D) Customer Mode 게이팅(카메라 not-ready면 Capture 차단)
**목표:** 테더링이 켜져 있고 카메라 상태가 ready가 아니면 `Start Capturing / Shoot`이 차단되는지 확인

중요:
- 현재 백엔드는 tethering이 켜져 있으면 `get_camera_status`가 기본적으로 `disconnected`를 반환합니다(Story 1.9에서 실제 ready로 바뀌도록 구현 예정).
- 따라서 이 시나리오는 “차단되는지”만 확인하면 됩니다.

Steps:
1. EDSDK Path를 올바르게 설정(시나리오 C 상태)
2. 앱을 Customer Mode로 부팅
   - Admin Settings → `General` → `Enable Customer Mode` 체크(필요 시) → 앱 재시작
3. Customer Mode에서 `Start Session` → 전화번호 뒤 4자리 입력(또는 빈칸) → `Continue`
4. Setup 화면 상단의 `Start Capturing` 버튼 확인
5. (가능하면) `Start Capturing`이 눌리지 않는 상태에서 안내 메시지 확인

Expected:
- `Start Capturing` 버튼이 비활성화(회색)
- “Camera not ready” 계열 안내 문구(작은 노란 글씨)

추가(선택):
- 이미 Capture 화면에 들어간 상태라면 `Shoot` 버튼이 `Camera Not Ready`로 비활성화

Pass:
- [ ] Setup에서 `Start Capturing`이 차단됨
- [ ] Capture에서 `Shoot`이 차단됨(가능 시)

증거(권장):
- [ ] Setup 화면 스크린샷 1장(버튼 비활성)
- [ ] Capture 화면 스크린샷 1장(버튼 비활성)

---

### 시나리오 E) 롤백(테더링 OFF 시 파일 기반 모드로 진행 가능)
**목표:** 테더링을 끄면(Customer gating이 카메라 준비 상태에 의존하지 않게) 진행 가능한지 확인

Steps:
1. Admin Settings → `Kiosk Configuration` → EDSDK Path 비움 → `Save Configuration`
2. 앱 재시작(권장: 상태 캐시/화면 상태를 깔끔히)
3. Customer Mode에서 세션 시작 → Setup 화면으로 진입

Expected:
- `Start Capturing` 버튼이 카메라 상태 때문에 비활성화되지 않음
- `Camera Diagnostics`에서 “manual mode” 안내

Pass:
- [ ] 테더링 OFF 시 차단이 풀림

증거(권장):
- [ ] Setup 화면 스크린샷 1장(버튼 활성)

## 9) 자주 겪는 문제(트러블슈팅)

### 빌드/실행이 안 될 때
- `npm start`에서 Rust/C++ 관련 에러가 나면:
  - Visual Studio Build Tools의 “Desktop development with C++” 설치 여부 확인
  - 새 터미널을 열어 환경 변수가 반영되게 한 뒤 재시도
- WebView2 관련 에러:
  - WebView2 Runtime 설치

### Customer Mode가 계속 “Unavailable”로 뜰 때
- 세션 루트 폴더 경로가 비어 있거나, 폴더가 실제로 존재하지 않으면 막힙니다.
  - `Kiosk Configuration`에서 `Session Root Path`를 **존재하는 폴더로 설정**하세요.

### EDSDK 검증이 계속 실패할 때
- EDSDK 루트 구조를 다시 확인:
  - `...\EDSDK\Dll\EDSDK.dll`
  - `...\EDSDK\Dll\EdsImage.dll`
- Header 경고가 싫다면 `Header\*.h`도 복사
