# Windows WM_DEVICECHANGE Camera Detection Manual Tests (Story 1.9.4)

Related story:
- `docs/stories/1.9.4.wm-devicechange-physical-connection-detection.md:1`

## Preconditions

- Windows 10/11 x64
- Boothy build with Story 1.9.4 implemented
- EOS 700D connected via USB cable (and optionally additional USB devices for noise testing)

## Scenarios

### A) App running -> plug camera
Steps:
1. Launch the app with the camera disconnected
2. Open Customer Mode Setup/Capture screen (or Admin Diagnostics) where camera status is visible
3. Plug in EOS 700D USB cable

Expected:
- Within 1s, UI transitions from `disconnected` -> `connected_not_ready` (or `connecting`) -> `ready` (if session auto-connect is enabled), or stays `connected_not_ready` until Admin connects (depending on chosen behavior)
- No manual refresh required

### B) App running -> unplug camera
Steps:
1. With the camera connected (and ideally `ready`), unplug the USB cable

Expected:
- Within 1s, UI transitions to `disconnected`
- Controller releases session resources safely (no crash/hang)
- Customer capture is gated/blocked immediately

### C) Camera already connected at app startup
Steps:
1. Close the app
2. Connect the camera
3. Launch the app

Expected:
- Initial status reflects "connected" without requiring a plug/unplug event

### F) Camera connected but EDSDK not configured
Steps:
1. Clear/invalid EDSDK path in Admin settings
2. Plug camera while the app is running (or launch the app with the camera already connected)

Expected:
- Status becomes `connected_not_ready` (not `ready`) and provides an Admin-actionable message ("configure EDSDK path" etc.)
- Customer UI remains customer-safe (no filesystem paths/raw SDK errors)

### D) Sleep / wake
Steps:
1. With camera connected, put Windows to sleep
2. Wake the machine

Expected:
- Status transitions to a safe state during sleep and recovers on wake (exact mapping depends on hardware behavior)
- Clear retry path exists in Admin if session does not auto-recover

### E) Noise filtering (optional)
Steps:
1. Plug/unplug unrelated USB devices (mouse/USB drive) while the app is running

Expected:
- Camera status does not flap unless the camera device itself changes
