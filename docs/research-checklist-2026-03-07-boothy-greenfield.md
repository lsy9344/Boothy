- [x] 이 체크리스트는 고객 여정, 운영 흐름, 정책, 문구, 지점 운영 표준화를 정교화하기 위한 문서다.

- [x] 다음 세션은 이 문서의 미완료 항목을 기준으로 질문을 이어간다.
- [x] 제품 하드웨어 전제는 키오스크가 아니라 GPU를 갖춘 고사양 데스크톱 PC와 모니터다.

## 0.5 현재 준비도 판단 (2026-03-07)

- 현재 문서는 제품 방향과 MVP 범위를 정하는 데에는 충분히 유효하다.
- 다만 이 상태를 바로 "모든 분야 전문가 킥오프용 최종 PRD"로 보기에는 아직 미완성이다.
- 지금 바로 착수 가능한 영역은 `리서치`, `UX 흐름 정교화`, `운영 정책 정리`, `카피 초안 작성`이다.
- 바로 착수하면 위험한 영역은 `아키텍처 확정`, `구현 범위 고정`, `지점 설정 모델 확정`, `운영자 권한 확정`이다.
- 미완료 항목은 모두 같은 우선순위가 아니다. 아래 순서로 먼저 채운다.

1. 실제 전화/원격지원 기준선과 실패 유형 비중
2. 고객 불안 포인트와 종료/인계 문구
3. 운영자 예외 처리 범위와 원격지원 시 필요한 상태 정보
4. 지점별 차이와 공통 설정의 분리 기준
5. 성공 지표와 측정 방식

- 현재 판단:
  - 제품 컨셉: 대체로 확정
  - 고객 여정 뼈대: 확정
  - 운영 표준/문구: 미확정
  - 지표/검증 방식: 미확정
  - 전문가 킥오프 준비도: `리서치/UX 착수 가능`, `본격 아키텍처/구현 킥오프는 보류`

## 1. 비즈니스 목표 검증

- [x] 핵심 문제는 소프트웨어 세팅/에러로 인한 전화 응대다.
- [x] 핵심 목표는 무인 운영 완성도 향상이다.
- [x] 현재 월간 전화/원격지원 건수 기준선 수집
  - 최근 1개월 전체 지점 합산 약 `500건`
- [x] 매장별 문의 유형 비중 수집
  - rough estimate 기준 `촬영 시작 지연 및 내보내기 지연 70%`, `쿠폰 종료 알림 혼선 20%`, `기타 10%`
- [x] 촬영 시작 실패 vs 종료 후 처리 실패 비중 재확인
  - `촬영 시작 지연`이 `내보내기 지연`보다 더 큼
- [x] 어떤 문제를 줄이면 운영자 체감이 가장 큰지 우선순위 재확인
  - `촬영 시작 지연`과 `내보내기 지연`의 중요도는 같음

확인된 운영 사실 (2026-03-07 추가):

- 가장 많은 문의는 `촬영 시작 지연`과 `내보내기 지연`이다.
- 문의 비중 rough estimate는 `촬영 시작 지연 및 내보내기 지연 70%`, `쿠폰 종료 알림 혼선 20%`, `기타 10%`다.
- `촬영 시작 지연`이 `내보내기 지연`보다 더 자주 발생한다.
- 다만 제품 우선순위는 `촬영 시작 지연`과 `내보내기 지연`을 동일하게 본다.
- `기타 10%`에서 가장 자주 반복되는 항목은 `예약문의`이며, `인원 추가 비용`, `구성`, `옵션` 문의가 중심이다.
- 주요 원인은 소프트웨어 오류로 인한 시작 지연 및 내보내기 지연이다.
- 고객이 실제로 자주 보게 되는 대표 오류 문구는 `프로그램 자동화 오류`다.
- 촬영실 입장 직후 사용방법을 몰라 촬영 프로그램을 실행하지 않은 상태에서 카메라 직결 리모컨을 먼저 누르는 상황이 발생한다.
- 이 경우 카메라와 PC의 테더링 연결이 준비되지 않은 상태에서 촬영이 먼저 진행되어, 고객은 촬영했다고 느끼지만 PC에 저장된 사진이 없을 수 있다.
- 촬영 프로그램 실행 전 리모컨 조작은 이후 세팅 병목으로 `카메라를 검색/감지하지 못하는 상태`로도 이어질 수 있다.
- 내보내기 자동화 오류가 나면 전 타임 고객 사진이 내보내지거나, `:50`에 정확히 내보내지 못해 다음 타임까지 지연이 전파된다.
- 기존 프로그램은 단순 매크로 성격이 강해, 라이트룸 내부 팝업/메뉴/포커스 상태가 달라지면 중간 단계에서 자동화가 막힌다.
- 즉, 고객은 `어떤 화면에서 끄는지`, `어떤 팝업이 떠 있는지`, `현재 어떤 창이 최상단인지`를 신경 쓰지 않아야 한다.
- 별도의 반복 문의 유형으로 `1시간 추가 쿠폰 사용 고객이 n시 50분 종료 알림을 보고 혼선`이 있다.
- 따라서 제품은 `오류 감소`와 함께 `연장 이용 고객의 시간 인지 혼선 제거`도 다뤄야 한다.
- 연장 이용 고객 여부는 향후 제품이 참조할 `구글시트의 쿠폰 사용 고객 정보`를 기준으로 구분한다.
- 비즈니스 컨텍스트 기준으로 `평일 1시간 추가 쿠폰` 고객의 시간 구조는 다음과 같다.
  - 촬영: `예약 시작 시각 + 1시간 40분`까지
  - 내보내기: `20분`
  - 셀렉실: 정시부터 `50분`
- 따라서 연장 고객에게는 시작 직후부터 `연장된 종료 시각`이 바로 보여야 한다.

## 2. 고객 여정 세부 질문

### 2.1 방문 직후

- [x] 초기 화면 중심 행동은 `PC에서 예약자명 + 휴대전화 뒤4자리 입력`
- [x] 고객이 입장 후 가장 먼저 헤매는 포인트 정리
  - `어떤 프로그램을 눌러야 하는가`보다 `지금 무엇부터 해야 하는가`를 먼저 헤맨다.
  - 따라서 첫 화면은 앱 이름 설명보다 `예약자명 + 뒤4자리 입력`이라는 첫 행동을 바로 드러내야 한다.
- [x] 고객이 실제로 읽는 안내 문구 길이의 한계 확인
  - 한 화면에는 `핵심 안내 1문장 + 보조 설명 1문장`까지만 허용한다.
  - 장문 절차 설명이나 2개 이상의 선택지를 동시에 읽게 하지 않는다.
- [x] 첫 화면에서 꼭 보여야 하는 안내 문구 최종 확정
  - `예약자명과 휴대전화 뒤4자리를 입력해 주세요.`
- [x] 고객이 `세팅 전 리모컨 조작`을 하지 않도록 시스템이 먼저 감지/방지하는 방향을 우선함
- [x] 고객이 라이트룸 내부 상태나 팝업 존재를 전혀 인지하지 않아도 되는 경험을 우선 목표로 함
- [x] 시스템으로 방지할 수 없을 때 어떤 경고문을 띄울지 정리
  - 보조 경고문 최종안: `화면에 촬영 가능이 표시될 때까지 리모컨을 누르지 마세요.`

### 2.2 세션 시작

- [x] 예약자명 + 휴대전화 뒤4자리 입력
- [x] 첫 체크인 화면 안내 문구는 `예약자명과 휴대전화 뒤4자리를 입력해 주세요.`
- [x] 첫 체크인 화면 보조 설명 문구는 `세션 확인용 정보입니다.`
- [x] 세션 시작 입력은 키오스크용 화면 키보드가 아니라 데스크톱 PC + 모니터 환경을 전제로 함
- [x] 예약자명은 필수 입력
- [x] 예약자명 입력칸은 한글만 입력 가능
- [x] 휴대전화 뒤4자리는 숫자 4자리 형식이 필수
- [x] 휴대전화 뒤4자리 입력칸은 숫자만 입력 가능
- [x] 입력 형식이 맞지 않으면 진행 차단
- [x] 입력 형식 오류 문구 최종안
  - `예약자명은 한글로 입력하고, 휴대전화 뒤4자리는 숫자 4자리로 입력해 주세요.`
- [x] 입력 형식 오류 문구는 `Enter` 전에도 빨간색으로 바로 노출
- [x] 체크인 화면이 열리면 커서는 처음부터 `예약자명` 입력칸에 자동으로 위치
- [x] 입력 형식이 맞으면 `Enter` 키만으로 바로 다음 단계로 이동
- [x] 체크인 화면에는 별도의 `다음` 버튼이 없음
- [x] 예약자명/휴대전화 뒤4자리가 실제 예약 정보와 일치하지 않아도 진행 차단하지 않음
- [x] 입력 직후 바로 프리셋 선택으로 이동
- [x] 세션명 형식은 `예약자명 + 휴대전화 뒤4자리`
- [x] 같은 날 동일 세션명이 다시 생기면 `_2`, `_3`을 붙여 구분
- [x] 전 타임 사진 혼선 자동 복구 예외 세션명은 `3월 2일 5시 예약손님` 같은 `월 일 시 예약손님`
- [x] 예외 자동 복구 세션명에는 `자동 복구` 같은 설명 문구를 붙이지 않음
- [x] 고객이 이 입력을 불편하게 느끼는지 확인
  - 현장 기준으로 고객은 `예약자명 + 휴대전화 뒤4자리` 입력을 대체로 무리 없이 수행한다.
- [x] 이름/뒤4자리 입력 이유를 화면에 설명할 필요가 있는지 검토
  - 짧은 보조 설명만 제공: `세션 확인용 정보입니다.`
- [x] 동명이인 충돌 시 고객에게 보이는 세션명은 `_2`, `_3`으로 구분

### 2.3 프리셋 선택

- [x] 첫 촬영 전 적용 프리셋은 정해져 있어야 함
- [x] 프리셋 선택 직후 바로 촬영 가능 화면으로 이동
- [x] 촬영 중 프리셋 변경 가능
- [x] 프리셋 변경은 바꾼 이후부터 촬영된 사진에만 적용
- [x] 프리셋 이름 체계 정리
  - 설명형 이름 사용: `웜톤`, `쿨톤`, `배경지 - 아이보리`, `배경지 - 핑크`
- [x] 고객이 이해하기 쉬운 프리셋 분류 방식 정리
  - `색감/톤`, `배경지`처럼 바로 이해 가능한 기준으로 묶기
- [x] 프리셋 예시 이미지 제작 기준 정리
  - 시스템에 현재 보이는 최신 실제 사진을 기준으로 사용
  - 각 프리셋은 같은 최신 사진에 적용된 결과로 비교
- [x] 프리셋 선택 화면 노출 개수는 최대 `6개`
- [x] 프리셋 노출 순서는 등록순
- [x] 프리셋 기본 선택은 세션과 무관하게 가장 마지막에 사용했던 프리셋으로 미리 맞춰둠
- [x] 고객은 미리 선택된 프리셋을 그대로 두고 별도 조작 없이 바로 촬영 가능 화면으로 넘어갈 수 있음
- [x] 매장별 추가 프리셋을 고객에게 어떻게 자연스럽게 섞어 보여줄지 검토
  - MVP에서는 매장별 추가 프리셋 경험 자체를 허용하지 않는다.
  - 전 지점이 같은 프리셋 카탈로그, 같은 설명형 이름, 같은 등록순 노출 규칙을 공유한다.

### 2.4 촬영 중

- [x] 고객 화면에는 자기 세션 폴더의 사진만 보여야 함
- [x] 고객은 자기 세션 폴더 안의 사진만 삭제 가능
- [x] 삭제 확정 시 사진은 즉시 영구 삭제
- [x] 삭제 완료 후에는 화면 중앙의 짧은 토스트로 `사진이 삭제되었습니다.` 문구를 잠깐 보여줘야 함

- [x] 가능하면 시스템이 3초 간격 촬영 신호를 대신 처리해 고객 조작을 없애는 방향이 우선
- [x] 위 처리가 가능할 때 촬영 가능 화면에는 `실제 촬영 종료 시각`만 표시
- [x] 위 처리가 불가능할 때만 촬영 가능 화면에 `리모컨 3초에 한 번`과 `실제 촬영 종료 시각` 표시
- [x] 남은 시간을 함께 보여줄 경우 보조 정보로 두고 `초 단위`까지 표시
- [x] 현재 적용 중인 프리셋 이름은 촬영 가능 화면에서 계속 보여주되 크게 강조할 필요는 없음
- [x] 기본 고객의 실제 촬영 종료 시각은 `n시 50분`
- [x] 고객을 가장 안심시키는 요소는 `방금 찍은 큰 사진`
- [x] 큰 사진 영역은 확대 보기 가능
- [x] 확대 상태에서도 삭제 버튼이 바로 보여야 함
- [x] 확대 상태에서는 사진을 이동하며 볼 수 있어야 함
- [x] 작은 썸네일 목록도 반드시 필요
- [x] 작은 썸네일 목록은 좌우 스크롤로 이전 사진까지 계속 확인 가능
- [x] 작은 썸네일 선택 시 큰 사진도 해당 사진으로 변경
- [x] 고객이 추가로 궁금해할 정보가 있는지 확인
  - 고객 화면에서 추가로 남겨둘 정보는 `실제 종료 시각`, `현재 프리셋`, `방금 찍은 사진` 정도면 충분하다.
  - 저장 경로, 내부 프로그램 상태, 운영자용 진단 정보는 고객 화면에 올리지 않는다.
- [x] 최근 사진 크게 보기 외에 필요한 안심 요소가 있는지 검토
  - `썸네일이 계속 쌓이는 것`, `자기 세션 사진만 보이는 것`, `실제 종료 시각이 안정적으로 보이는 것`을 보조 안심 요소로 본다.
- [x] 촬영 중 썸네일은 좌우 스크롤로 제공

### 2.5 종료 직전

- [x] 고객 유형별 실제 종료 시각 기준 경고/종료 알림 필요
- [x] 종료 직전 경고 카피 초안 작성
  - `촬영 종료까지 5분 남았습니다. 필요한 사진을 마무리해 주세요.`
- [x] 종료 알림 카피 초안 작성
  - `촬영이 종료되었습니다. 내보내기를 시작합니다. 잠시만 기다려 주세요.`
- [x] 고객이 종료를 갑작스럽지 않게 느끼도록 하는 문구 정리
  - 실제 종료 시각 5분 전에 먼저 경고를 보여준다.
- [x] 운영자 예외 연장이 적용되면 종료 5분 전 경고와 종료 2분 전 전화 전환 기준도 새 종료 시각 기준으로 다시 계산해야 함
- [x] 종료 후 대기 경험을 납득시키는 표현 방식 정리
  - 대기 상태는 `멈춤`처럼 보이면 안 되고, `지금 내 사진을 준비 중`이라는 해석이 가능해야 한다.
  - 기본 표현은 진행률 + 짧은 안심 문구 + 완료 후 즉시 세션명/다음 행동 제시다.
- [x] 1시간 추가 쿠폰 사용 고객은 `n시 50분`이 아니라 `예약 시작 시각 + 1시간 40분`에 종료 알림이 떠야 함
- [x] 1시간 추가 쿠폰 사용 고객은 시작 시점부터 연장 종료 시각이 바로 보여야 함
- [x] 운영자 예외 연장이 적용되면 고객 화면의 종료 시각도 즉시 갱신되어야 함
- [x] 운영자 예외 연장이 적용되면 고객에게 시간 연장 안내 문구도 보여야 함
- [x] 1시간 추가 쿠폰 사용 고객 종료 알림 카피 초안 작성
  - 일반 고객과 동일한 카피를 사용하되, 노출 시각은 실제 종료 시각 기준으로 적용

- [x] `내보내기 중` 예외 연장이 적용되면 현재 내보내기를 끝낸 뒤 촬영 가능 화면으로 복귀
- [x] `내보내기 중` 예외 연장 후 추가 촬영 결과물은 기존 세션 폴더에 계속 저장

## 3. 고객 불안 포인트 정리

- [x] 고객이 가장 많이 불안해하는 순간 Top 5 정리
  - `촬영 준비 중` 상태가 길어져 시작이 늦어질 때
  - `촬영 가능` 표시 전에 리모컨을 눌러 저장이 안 됐을까 불안할 때
  - 촬영 중 최근 사진과 썸네일이 갱신되지 않아 정상 저장 여부가 흔들릴 때
  - 종료 후 내보내기가 늦어 다음 단계로 못 넘어갈 때
  - 연장 고객인데 실제보다 이른 종료 알림을 보거나, 다른 사람 사진이 나갈까 불안할 때
- [x] 각 불안 포인트별 제품 대응 문구 정리
  - 시작 준비 지연: `촬영 준비 중입니다. 잠시만 기다려 주세요.`
  - 카메라 확인 지연: `카메라를 확인하고 있습니다. 잠시만 기다려 주세요.`
  - 저장 이상: `촬영한 사진이 저장되지 않고 있습니다. 전화해 주세요.`
  - 내보내기 진행: `내보내는 중입니다. [몇 %]`
  - 내보내기 장기 지연: `내보내기가 지연되고 있습니다. 전화해 주세요.`
- [x] 고객이 전화하게 되는 대표 상황과 화면 연결
  - `촬영 준비 중입니다. 잠시만 기다려 주세요.` 상태가 실제 종료시간 2분 전까지 지속될 때
  - `촬영한 사진이 저장되지 않고 있습니다. 전화해 주세요.` 화면으로 전환될 때
  - `내보내기가 지연되고 있습니다. 전화해 주세요.` 화면으로 전환될 때
  - 전 타임 사진 혼선이 자동 복구되지 않는다고 판단될 때
- [x] 고객이 "정상 동작 중"임을 인지하는 기준 정리
  - 시작 전: `촬영 준비 중` 또는 `촬영 가능` 중 하나의 명확한 상태만 보인다.
  - 촬영 중: 방금 찍은 큰 사진과 썸네일이 계속 갱신된다.
  - 종료 후: 진행률이 보이거나, 완료 후 세션명과 다음 행동이 즉시 보인다.

현재까지 확인된 대표 불안 상황:

- `프로그램 자동화 오류`가 떠서 촬영을 시작하지 못할 때
- 촬영 가능 표시 전에 리모컨을 눌렀는데, 실제로는 저장된 사진이 없을까 불안할 때
- 소프트웨어 오류로 촬영 시작이 지연될 때
- 내보내기가 늦어져 다음 단계로 못 넘어간다고 느낄 때
- 내보내기 자동화 오류로 내 사진이 아닌 다른 사람 사진이 나갈까 불안할 때
- `1시간 추가 쿠폰`을 사용했는데 실제 종료 시각보다 이른 `n시 50분 종료 알림`이 떠서 시간이 끝난 줄 알 때

  - `내보내는 중입니다. 시간이 조금 더 걸리고 있습니다. 잠시만 기다려 주세요. [몇 %]`
- [x] 종료시간 2분 전까지 완료되지 않았을 때의 전화 안내 문구 최종안
  - `내보내기가 지연되고 있습니다. 전화해 주세요.`
- [x] 전화 안내 화면의 전화번호 정책 확정
  - 지점별 번호 사용
  - 미설정 시 기본값 `010-0000-0000`
- [x] `사진저장폴더` 이동 안내 문구 최종안
  - `내보내기가 완료됐습니다. 셀렉실로 이동하여 '바탕화면'의 '사진저장폴더'에서 [세션명]을 열어 셀렉 및 인화를 시작하세요.`

## 7. 자동화 우선순위 정리

- [x] 자동화는 고객 조작을 줄이는 방향이어야 함
- [x] 지금 당장 가장 큰 가치가 있는 자동화 3개 재정렬
  1. `촬영 시작 준비 상태 자동 판정과 시작 진입 자동화`
  2. `고객 유형별 실제 종료 시각 계산과 경고/종료 알림 자동화`
  3. `내보내기 결과물 준비와 세션 인계 자동화`
- [x] MVP 이후 2단계 자동화 후보 정리
  - `예약 검증`
  - `구글시트 기반 쿠폰/고객 구분`
  - `지점별 운영 설정 중앙관리`
  - `사전 안내/후기 요청 자동화`
- [x] 예약/구글시트 연동이 실제로 어떤 운영 문제를 먼저 해결하는지 명확화
  - 첫 번째 해결 대상은 `1시간 추가 쿠폰 고객의 연장 종료 시각 오표시`다.
  - 두 번째 해결 대상은 `다음 타임 충돌 확인`과 `연장 고객 시간 규칙 자동 반영`이다.
- [x] 예약/구글시트 연동의 첫 번째 제품 가치는 `1시간 추가 쿠폰 고객의 종료 시각 반영`임
- [x] 연장 이용 고객에게 일반 종료 알림이 뜨지 않도록 전체 시간 경험 정리
  - 세션 시작 직후부터 `연장된 촬영 종료 시각`을 바로 보여준다.
  - 종료 5분 전 경고, 종료 알림, 내보내기 대기 기준도 모두 실제 연장 종료 시각 기준으로 계산한다.
  - 실제 종료 시각이 아닌 `n시 50분` 일반 알림은 연장 고객에게 노출하지 않는다.
- [x] 촬영 가능 여부는 가능하면 팝업/포커스/매크로 진행 상태가 아니라 시스템이 직접 판단해야 함
- [x] 경고문은 시스템 방지가 불가능할 때의 차선책으로만 사용

## 8. 성공 지표 보강

- [x] 핵심 지표는 전화/원격지원 감소
- [x] 보조 지표 정의
  - MVP 성공 판단의 필수 KPI는 `월 전화/원격지원 250건 이하` 하나
  - 추가 지표는 필수가 아니라 참고용 선택 지표
- [x] 고객 혼자 촬영 시작 성공률 측정 방식 정의
  - 세션 단위로 측정한다.
  - 정의: `체크인 이후 운영자 전화/원격지원 개입 없이 첫 사진 저장이 발생한 세션 비율`
- [x] `사진저장폴더` 인계 성공률 측정 방식 정의
  - 세션 단위로 측정한다.
  - 정의: `내보내기 완료 후 고객 세션명이 사진저장폴더에 정상 생성되어 추가 문의 없이 셀렉실 이동이 가능한 세션 비율`
- [x] 종료 후 대기시간 측정 방식 정의
  - 세션 단위로 측정한다.
  - 정의: `실제 촬영 종료 시각부터 완료 화면 또는 전화 안내 화면이 뜨기까지의 경과 시간`
  - 현장 가정: 고객은 종료 5분 전부터 시간을 계속 확인하며 불안이 커지기 시작한다.
- [x] `1시간 추가 쿠폰 고객의 종료 알림 혼선 문의`를 별도 추적할지 정의
  - 별도 추적하지 않음
  - `촬영 시작 전 실제 종료 시각 표시`로 해결하는 문제로 봄

## 9. 다음 세션에서 먼저 물어볼 질문 후보

- [x] 고객이 가장 많이 전화하는 실제 문구/에러 상황은 무엇인가
  - `프로그램 오류`
  - `촬영이 안돼요`
  - `내보내기 했는데 셀렉실에서 안보여요`
  - `내보내기 중 컴퓨터가 멈췄어요`
- [x] 프리셋을 고객 눈높이에서 어떻게 이름 붙일 것인가
  - 추상적 무드명보다 설명형 이름 사용: `웜톤`, `쿨톤`, `배경지 - 아이보리`, `배경지 - 핑크`
- [x] 완료 화면에서 고객이 반드시 기억해야 하는 정보는 `세션명` 하나
- [x] 고객 진행 화면에서 예외 연장 외에 별도 운영자 기능은 없음
- [x] 매장별로 달라도 되는 경험과 절대 같아야 하는 경험은 무엇인가
  - 절대 같아야 하는 경험: 고객 여정, 핵심 문구, 프리셋 카탈로그, 촬영/종료/인계 흐름, 경고와 전화 전환 기준
  - 달라도 되는 항목: 고객에게 노출되는 전화번호 하나
- [x] 1시간 추가 쿠폰 고객 화면에는 `촬영 종료 시각`만 크게 보여주고 `전체 이용 종료 시각`은 노출하지 않음
- [x] 1시간 추가 쿠폰 고객에게 `쿠폰 적용됨` 같은 별도 문구는 노출하지 않고, 연장된 종료 시각만 보여줌
- [x] 고객에게 `세팅 전 리모컨 조작 금지`를 먼저 요구하지 않고, 그런 판단이 필요 없는 경험을 우선 목표로 함
- [x] 촬영 가능 상태는 가능하면 라이트룸 매크로 성공이 아니라 카메라 준비 상태 기준으로 판단해야 함
- [x] `촬영 준비 중입니다. 잠시만 기다려 주세요` 상태에서는 고객 조작을 완전히 막음
- [x] 시스템으로 방지할 수 없을 때 고객에게 보여줄 `세팅 전 리모컨 조작 금지` 문구 확정

## 10. 완료 기준

- [x] 고객 여정 각 단계별 핵심 문구 확정
  - 시작 준비: `촬영 준비 중입니다. 잠시만 기다려 주세요.`
  - 카메라 확인: `카메라를 확인하고 있습니다. 잠시만 기다려 주세요.`
  - 체크인: `예약자명과 휴대전화 뒤4자리를 입력해 주세요.`
  - 체크인 보조: `세션 확인용 정보입니다.`
  - 형식 오류: `예약자명은 한글로 입력하고, 휴대전화 뒤4자리는 숫자 4자리로 입력해 주세요.`
  - 보조 경고: `화면에 촬영 가능이 표시될 때까지 리모컨을 누르지 마세요.`
  - 종료 5분 전: `촬영 종료까지 5분 남았습니다. 필요한 사진을 마무리해 주세요.`
  - 종료 시: `촬영이 종료되었습니다. 내보내기를 시작합니다. 잠시만 기다려 주세요.`
  - 내보내기 진행: `내보내는 중입니다. [몇 %]`
  - 진행 정체: `내보내는 중입니다. 시간이 조금 더 걸리고 있습니다. 잠시만 기다려 주세요. [몇 %]`
  - 저장 이상: `촬영한 사진이 저장되지 않고 있습니다. 전화해 주세요.`
  - 저장 경로 이상: `사진 저장 준비에 문제가 있습니다. 전화해 주세요.`
  - 완료: `내보내기가 완료됐습니다. 셀렉실로 이동하여 '바탕화면'의 '사진저장폴더'에서 [세션명]을 열어 셀렉 및 인화를 시작하세요.`
- [x] 운영자 모드 목적과 권한 범위 확정
  - 목적: 고객 진행을 끊지 않고 예외 상황만 빠르게 안정화하는 것
  - 고객 진행 화면 직접 권한: `시간 연장`만
  - 운영자 세팅 화면 권한: 상태 확인, 재시작, 내보내기 재시도, 세션 분리/복구, 최근 오류 확인
  - 가격/환불/예약 예외 같은 상업 정책은 제품 밖 운영 권한으로 본다.
- [x] 지점 공통 규칙과 지점별 설정 항목 분리
  - 공통 규칙: 전화번호 외 전체
  - 지점별 설정: 전화번호
- [x] MVP 이후 확장 항목과 현재 범위 명확 분리
  - 현재 범위 밖: 예약문의 대응 기능
  - 예시: `인원 추가 비용`, `구성`, `옵션` 문의
- [x] 다음 세션에서 바로 세부 기능 명세로 내려갈 수 있을 정도로 제품 요구사항 구체화
  - 현재 PRD는 고객 여정, 핵심 카피, 운영자 권한, 예외 처리, 자동화 우선순위, 보조 지표 정의까지 포함해 세부 기능 명세 단계로 내려갈 수 있다.
  - 남은 것은 문서 구조 보완이 아니라 실제 운영 로그와 현장 관찰로 검증하는 후속 리서치다.
