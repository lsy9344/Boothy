# 제품 요구사항 문서(PRD) v1.1 — (가칭) RapidTetherRAW Customer Mode (RapidRAW Fork)

문서 버전: v1.1  
작성일: 2026-01-02  
대상 릴리스: MVP → 1.0 (무인 셀프 스튜디오 Customer Mode 포함)  
작성자: Product Owner / Engineering  

> 본 문서는 기존 PRD v1.0(2025-12-30)의 “전문가/운영자 중심 경량 테더+현상+Export 도구” 범위를 유지하되, **무인 셀프 스튜디오** 운영에 필요한 **Customer Customer Mode Mode / Admin Mode / 세션 타이머 / Export 게이트 / 프라이버시 리셋 / 장애 UX / 스마트 Export 파이프라인**을 추가 반영한 통합안이다.

---

## 1. 개요(Executive Summary)
RapidTetherRAW는 **Canon EOS 700D 단일 기종**을 대상으로 하는 **Windows 전용 테더 촬영 + RAW 현상 + Export** 데스크톱 앱이다.  
Lightroom Classic에서 사용하던 작업 중 “**카메라 연결(테더) / 라이브뷰 / 촬영 후 즉시 표시 / 캡처 프리셋(필터) 적용 / Export**”만을 **초미니멀 UX**로 제공한다.

본 제품은 **RapidRAW(AGPL-3.0) 코드를 활용하는 포크(fork)**를 전제로 한다. RAW 현상 엔진/보정 파라미터/프리셋/비파괴(사이드카) 워크플로우는 RapidRAW 기반이며, 테더링은 **Canon EDSDK(사용자 로컬 설치/경로 설정)**로 추가 구현한다.

### 1.1 제품 포지셔닝(두 가지 모드)

> **용어 정리(중요)**: 본 문서에서 말하는 *Customer Mode*는 **실제 키오스크 하드웨어/OS 쉘 교체**를 전제하지 않는다.  
> 즉, **일반 Windows PC에서 실행되는 전용 앱(Full-screen + 최소 조작 + 강제 플로우)** 컨셉이다.  
> 다만 무인 운영 안정성을 위해 OS 제어 수준을 선택할 수 있도록 **UI Lock-down 레벨(0~2)**을 옵션으로 정의한다.

- **Customer Customer Mode Mode(기본)**: 무인 셀프 스튜디오 고객이 사용. “예약 확인 → 필터 선택 → 촬영 → 자동 Export/전송 → 자동 리셋”의 **단일 경로(상태머신)**만 제공.
- **Admin Mode(PIN)**: 매장 운영자/관리자용. 카메라 고정 세팅, 프리셋 관리, Export 규칙, 장애 로그/진단, 테스트 촬영 등 운영 기능 제공.

## 2. 제품 목표와 비전(Goals & Vision)
### 2.1 비전
무인 셀프 스튜디오에서 고객이 “컴퓨터를 조작한다”는 느낌 없이, **스티커 사진기/스마트 거울처럼** 간단하게  
“촬영 → 즉시 확인(PC) → 동일 보정(필터/프리셋) 연속 적용 → 자동 Export/수령”을 경험하게 한다.

### 2.2 목표
- **EOS 700D 테더 워크플로우 최적화**: 연결 안정성, 라이브뷰, 촬영 후 즉시 반영(Preview).
- **고객 실수 제거**: 세션명 입력/폴더 생성/내보내기 버튼 클릭 등 운영 행동을 고객 UI에서 제거.
- **강제적이고 자동화된 흐름(Railroaded Workflow)**:
  - 세션 시간(예: 50분) 종료 시 **촬영 잠금 + ExportLock(게이트)**로 강제 전환
  - Export가 끝나기 전에는 다음 세션 시작 불가
- **캡처 프리셋 개념 유지**:
  - 선택된 “캡처 프리셋(필터)”은 **이후 촬영분부터 자동 적용**
  - 프리셋 변경 시 **변경 이후 촬영분부터 적용**
  - 각 사진은 개별 보정값을 비파괴로 저장(사이드카)

### 2.3 비목표(의도적으로 하지 않는 것)
- 다중 카메라/다중 제조사 지원
- 방대한 라이브러리/태깅/레이팅/검색/AI 기능/파노라마 등 Lightroom의 전체 기능
- EDSDK 바이너리/런타임을 앱에 동봉(재배포하지 않음)
- 완전한 예약/결제 시스템 자체 구현(단, **예약 식별자 입력/조회 인터페이스**는 제공 가능)

## 3. 타겟 사용자 및 시나리오(Personas & Scenarios)
### 3.1 타겟 사용자(운영자/관리자)
- 셀프 스튜디오 운영자 1인 또는 소수 인원
- EOS 700D를 고정 세팅으로 쓰며, 촬영 품질/세팅 고정/장애 대응을 책임진다.

### 3.2 타겟 사용자(고객: 무인 매장)
- 사진 촬영/PC 조작에 익숙하지 않은 일반 사용자
- “예약 확인 후 1시간 동안 촬영하고 결과물을 받아가는” 고객

### 3.3 핵심 시나리오(End-to-End) — Customer Customer Mode Mode
> 기본 운영 타임박스 예시: **50분 촬영 + 10분 정리/Export(또는 백그라운드 처리)**

**상태머신(단일 경로)**
1) **Idle(대기)**: “예약번호/QR 스캔”만 노출, 카메라/디스크/이전 세션 상태 자동 점검  
2) **Setup(세션 시작/필터 선택)**: 예약 정보 확인 → 자동 폴더 생성 → 필터(캡처 프리셋) 선택  
3) **Capture(촬영)**: 큰 타이머 표시, 촬영/끝내기 버튼만 제공(슬라이더/고급 기능 숨김)  
4) **ExportLock(자동 내보내기/전송, 촬영 잠금)**: 촬영 종료 시 **강제 전환**, Export 진행률 표시  
5) **Complete(완료/수령 안내) → Reset(자동 초기화)**: QR/이메일 등 수령 안내 후 자동 로그아웃/리셋

### 3.4 핵심 시나리오(End-to-End) — Admin Mode
1) PIN 입력으로 Admin Mode 진입  
2) 카메라 고정 세팅/잠금 정책 설정(ISO/셔터/조리개/WB 등)  
3) 프리셋(필터) 관리(추가/삭제/정렬/미리보기 생성)  
4) Export 규칙(품질/리사이즈/파일명/출력 경로/클라우드 전송 옵션) 설정  
5) 장비 점검(카메라 연결, 디스크 여유, 테스트 촬영, 로그 확인)

## 4. 제품 범위(Scope)
### 4.1 In-Scope (MVP/1.0)
#### 4.1.1 테더링(EDSDK)
- 지원 기종: **Canon EOS 700D만 지원**
- 연결/해제/재연결
- **라이브뷰**: Customer Mode에서는 메인 화면 내 임베드(우선), Admin에서는 팝업 가능
- 촬영 트리거:
  - 기본: 앱이 셔터 트리거(권장) — 고객 입력(큰 버튼/터치) → 앱 → EDSDK 셔터
  - 보조: 물리 리모컨 촬영 감지(가능하면 지원하되, Customer Mode 운영 안정성을 위해 제한/비권장 옵션화)
- 저장 정책: **PC(Host)로만 저장**
- 촬영 파일 수신 후 자동 반영:
  - 세션 폴더에 저장
  - UI에 즉시 로드(최근 촬영본 자동 선택)
  - 선택된 “캡처 프리셋” 자동 적용(이후 촬영분에도 유지)

#### 4.1.2 RAW 현상(보정)
RapidRAW의 현상 기능 중 아래 항목을 제공한다(엔진은 유지하되, Customer Mode에서는 대부분 숨김).
- 화이트밸런스: Temp/Tint
- 톤: Exposure/Contrast/Highlights/Shadows/Whites/Blacks
- 컬러: Vibrance/Saturation
- 디테일: Sharpening/Noise Reduction(휘도/색상)
- 변형: Crop/Rotate/Flip (Admin 전용 또는 제한 노출)
- 톤 커브: Luma 및 RGB 커브(가능 범위는 RapidRAW 구현에 따름) (Admin 전용)
- 프리셋:
  - 캡처 프리셋(테더 촬영에 자동 적용되는 프리셋)
  - 사용자 프리셋 저장/불러오기/적용
- 히스토그램/클리핑 표시(가능하면, RapidRAW 기반) (Admin 전용 또는 옵션)
- Undo/Redo(비파괴 편집) (Admin 전용 또는 옵션)

#### 4.1.3 비파괴 저장(사이드카)
- 각 RAW 파일 옆에 **편집 값(sidecar)**을 저장한다(예: RapidRAW의 `.rrdata` 방식 또는 호환 포맷).
- 기존 사진을 다시 열면 저장된 편집 값이 복원된다.
- “캡처 프리셋”은 **새로 들어오는 촬영본의 초기 값**으로만 적용되며, 과거 촬영본의 값은 자동으로 바뀌지 않는다.

#### 4.1.4 내보내기(Export) — Smart Export Pipeline 포함
- 출력 포맷: JPEG(필수), 기타 포맷은 차기 고려
- 단일/다중 선택/세션 전체 Export
- 파일명 규칙(템플릿) 및 출력 폴더 선택
- **Smart Export Pipeline(핵심)**:
  - 촬영 중 유휴 시간에 **백그라운드로 프리셋 적용된 JPEG(임시 또는 최종)를 선생성**(큐 기반)
  - 세션 종료 시점에는 “남은 항목”만 처리하여 대기 시간을 최소화
  - 큐가 밀리면 UI에 “대기 n장”을 표시하고 정책에 따라 촬영 속도를 제한할 수 있음(옵션)

#### 4.1.5 세션 관리(무인 운영 최적화)
- Customer Mode: 세션 폴더/세션명은 고객이 입력하지 않음(자동 생성)
- 기본 폴더명 규칙 예시: `YYYYMMDD_HH00_예약ID_이름(선택)`
- 세션 내 이미지 목록: 최근 촬영 순 최소 UI(필름스트립)
- **세션 종료 후 리셋/정리**:
  - 고객 UI에서 이전 세션 접근 불가
  - 필요 시 자동 삭제/보관 정책(예: 30일 보관, 디스크 임계치 시 오래된 세션부터 삭제) — Admin 설정

#### 4.1.6 Customer Mode Mode(고객용 전체화면) + Admin Mode(PIN)
- Windows 작업표시줄/창 닫기/Alt+Tab 등 **일부 시스템 인터랙션 제한**(가능 범위 내)
- Customer Mode에서 노출되는 컨트롤은 최소화:
  - 예약 확인/필터 선택/촬영/종료/수령 안내 정도
  - 현상 슬라이더/파일 탐색/Export 옵션 상세는 숨김
- Admin Mode에서만 고급 설정/진단/수동 Export 옵션 노출

#### 4.1.7 카메라 세팅 고정(Lock) & 가드레일
- 세션 시작 시 지정된 세팅(모드/ISO/셔터/조리개/WB 등)을 EDSDK로 **강제 적용**
- 세션 중 주기적 감지(폴링) 또는 이벤트 기반 감지로 변경을 탐지해 **자동 복구**
- 연속 변경 감지 시:
  - 경고 UI → 필요 시 Capture 비활성화(안전 모드)
- (권장) 물리적 조작 방지를 위한 하우징/케이지는 운영 가이드로 제안(앱 외 범위)

#### 4.1.8 타임 매니지먼트(Session Timer) & Export 게이트
- 세션 시작과 동시에 타이머 실행(예: 50분 Capture)
- **10/5/1분 전** 화면+사운드(음성 포함 가능) 안내
- 50분 도달 시:
  - Capture 비활성화(셔터 트리거 차단)
  - **ExportLock 화면으로 강제 전환(고객 거부 불가)**
- Export 미완료 상태에서는 다음 세션 시작 불가

#### 4.1.9 장애 UX / 헬프 플로우
- Idle/Setup 단계에서 자동 점검:
  - 카메라 연결, EDSDK 경로 유효성, 저장 경로 권한, 디스크 여유, 이전 세션 Export 미완료 여부
- 문제 발생 시 고객에게는:
  - “진행 불가” + “딱 1가지 행동(도움 요청)”만 제공(연락처/원격지원 코드)
- Admin에는 상세 로그/원인/재시도 버튼 제공

### 4.2 Out-of-Scope (v1 비범위)
- 다중 카메라 지원 / 카메라 선택 UI
- 고급 라이브러리 기능(폴더 트리 관리, 태깅, 레이팅, 검색)
- 로컬 마스킹/브러시, AI 마스킹/생성형 편집, 파노라마/스티칭
- 렌즈 보정/퍼스펙티브 워핑 등 고급 변형(필요 시 차기)
- Lightroom XMP 라운드트립(읽기/쓰기 호환)
- 앱 자체의 결제/예약 관리 시스템(단, 예약 식별자 입력/조회 API 연동은 가능)
- 클라우드 전송의 결제/계정/구독 기능(전송 자체는 옵션으로만)

## 5. 사용자 경험(UX) 요구사항
### 5.1 Customer Mode UX 원칙(무인 최적화)
- 고객이 할 일은 **최대 2개**:
  1) 예약 확인(QR/예약번호)
  2) 촬영(및 필요 시 필터 선택)
- “다음에 무엇을 눌러야 하는지”를 고객이 판단하지 않도록 **상태머신으로 UI를 제한**
- **알림이 아닌 강제 전환**:
  - 시간 종료 시 Capture 화면 자체를 종료하고 ExportLock으로 이동
- 프라이버시 기본:
  - 세션 종료 후 이전 사진 UI 접근 불가
  - 파일 탐색/저장 경로 노출 금지

### 5.2 Customer Mode 화면 구성(최소 5화면)
1) **Idle**: 예약번호/QR 안내 + 장비 상태(간단 아이콘)  
2) **Setup**: 예약 확인 + 필터 선택(카드/칩) + “시작”  
3) **Capture**: 큰 타이머 + 최근 사진/라이브뷰 + “촬영/끝내기”  
4) **ExportLock**: “변환/전송 중” 진행률 + 도움 요청  
5) **Complete/Reset**: 수령 안내(QR/이메일) + 자동 초기화 카운트다운

### 5.3 Admin 화면 구성(요약)
- 카메라 설정(잠금 정책 포함)
- 프리셋 관리(리스트/미리보기/정렬)
- Export 규칙 설정
- 시스템/장비 점검(디스크/경로/테더 상태)
- 로그/리포트/테스트 촬영

### 5.4 UX 원칙(공통)
- 에러는 **짧고 구체적으로**
- 진행 상태(전송/Export/큐 대기)는 반드시 시각화
- 라이브뷰/촬영 동시성은 바디 특성을 감안해 자동 재개/명확한 상태 표시

## 6. 기능 요구사항(Functional Requirements)
### 6.1 모드/권한
- 기본은 Customer Mode Mode로 부팅
- Admin Mode는 PIN으로만 진입(설정 가능)
- Customer Mode에서는 파일 시스템/고급 현상 패널/Export 옵션 상세를 숨김

### 6.2 세션 시작(예약 확인/자동 폴더)
- Customer Mode 시작 입력:
  - QR 스캔(가능 시) 또는 예약번호/휴대폰 뒷자리 입력
- 예약 정보 소스(택1 또는 병행):
  - 로컬 DB(JSON/SQLite)
  - 외부 예약 시스템 API(선택)
- 세션 폴더명 규칙은 Admin에서 설정 가능
- 세션 시작 시:
  - 카메라 연결/EDSDK 경로/저장 경로/디스크 여유 점검
  - 실패 시 “진입 불가 + 도움 요청” 화면

### 6.3 테더 연결/세션
- 세션 폴더 준비 없이 테더 연결을 시작할 수 없다(Customer Mode)
- 연결 성공 시:
  - 저장 위치 “PC(Host)” 강제
  - 촬영 이벤트 핸들러 등록(Object/Property/State)
- 연결 해제 시:
  - 핸들러/세션 정리, 라이브뷰 종료, UI 초기화

### 6.4 촬영(앱 트리거 우선) & 셔터 차단
- Customer Mode의 촬영은 앱 트리거 우선(권장)
- 리모컨 촬영 감지는 옵션(운영 정책에 따라 off 가능)
- 타이머 만료/ExportLock 상태에서는:
  - 앱 촬영 버튼 비활성화
  - 셔터 트리거 호출 차단
  - (가능한 범위) 리모컨 이벤트는 무시/큐잉 정책 적용(명시)

### 6.5 카메라 세팅 Lock(가드)
- 세션 시작 시 지정 값 강제 적용
- 세션 중 변경 감지 시 자동 복구(주기/이벤트)
- 변경이 반복되면:
  - 경고 → 필요 시 Capture 잠금(안전 모드)
- UI에 “세팅 정상/변경 감지/복구 중” 상태를 표시(Customer Mode는 단순, Admin은 상세)

### 6.6 이미지 로드/보정/사이드카
- RAW 로드 시:
  - sidecar 존재하면 적용
  - 없으면 현재 캡처 프리셋으로 초기 값 생성 + sidecar 생성
- Customer Mode에서는 프리셋/강도(선택) 정도만 노출(슬라이더 숨김)
- Admin에서는 슬라이더/커브/Undo 등 제공 가능

### 6.7 프리셋(필터)
- Customer Mode 필터 UI:
  - 카드/칩 + 미리보기(썸네일) 기반
  - 최대 N개(예: 6~12개)로 제한(설정)
  - (선택) 강도 1개 슬라이더만 허용
- Admin:
  - 프리셋 추가/삭제/정렬/미리보기 생성
  - “캡처 프리셋”과 “일반 프리셋” 개념 유지

### 6.8 Smart Export Pipeline(백그라운드)
- 촬영 파일 수신 후:
  - 즉시 프리뷰 로드
  - 백그라운드 큐에 JPEG 생성 작업 enqueue
- 세션 종료 시:
  - ExportLock에서 큐 drain(남은 항목 처리)
  - 완료 후 Complete 화면으로 자동 전환
- 큐 상태 표시:
  - Customer Mode: “처리 중(남은 n장)” 정도
  - Admin: 상세(작업 시간/실패 항목/재시도)

### 6.9 ExportLock(게이트) & 완료/전송
- 시간 만료 또는 고객이 “끝내기” 누르면 ExportLock 진입(강제)
- ExportLock에서 가능한 동작:
  - 진행률 보기
  - 도움 요청(연락처/원격 코드)
- 완료 후:
  - 수령 안내(옵션: QR 다운로드/이메일 전송/로컬 출력 폴더 등)
  - 자동 로그아웃/초기화(카운트다운)
- (옵션) 클라우드 전송:
  - v1에서는 “전송 모듈 플러그인/웹훅 호출” 수준으로 제한

### 6.10 프라이버시/리셋/청소
- Customer Mode에서 파일 탐색기 접근 방지(가능 범위)
- 세션 종료 시:
  - UI에서 이전 세션 사진 제거
  - 임시 캐시/프리뷰 삭제
- 보관/삭제 정책은 Admin에서 설정(기간/디스크 임계치)

### 6.11 장애/진단
- 자동 점검 항목: 카메라, EDSDK 경로, 저장 권한, 디스크, 큐 상태, 이전 Export 미완료
- 실패 시:
  - Customer Mode: 단순한 안내 + 도움 요청
  - Admin: 오류 코드/로그/재시도/리포트 내보내기

## 12. 리스크 및 완화(Risks & Mitigations)
- **EDSDK EULA/배포 제약**: 사용자 설치 + 경로 설정 UX 강화
- **리모컨 촬영 차단의 한계**: Customer Mode는 앱 트리거 우선 + 리모컨 옵션화
- **백그라운드 JPEG로 인한 성능 저하**: 큐 우선순위/스로틀링/대기 표시
- **무인 프라이버시 사고**: 세션 격리/자동 리셋/파일 탐색 차단 강화
- **업스트림 유지보수**: RapidRAW 특정 릴리스/커밋에 핀(pin)하고 기능 추가 최소화

## 13. 수용 기준(Acceptance Criteria)
### 13.1 Customer Mode 운영 수용 기준
1) Customer Mode가 기본으로 실행되며, 고객은 예약 확인 없이 Capture에 진입할 수 없다.  
2) 세션 시작 시 폴더가 자동 생성되고, 고객이 폴더명/경로를 입력하지 않는다.  
3) 세션 타이머가 동작하며 10/5/1분 경고가 출력된다.  
4) 50분 도달 시 Capture가 비활성화되고 ExportLock으로 강제 전환된다.  
5) Export가 완료되기 전에는 다음 세션 시작이 불가하다.  
6) 세션 종료 후 이전 사진/세션이 고객 UI에서 접근 불가하며 자동 리셋된다.

### 13.2 테더/현상/Export 수용 기준
7) Windows에서 EOS 700D 연결/해제/재연결이 안정적으로 동작한다.  
8) 촬영 시 RAW가 PC 세션 폴더로 저장되고 UI에 자동 표시된다.  
9) 캡처 프리셋은 이후 촬영분에 자동 적용되며, 프리셋 변경 이후 촬영분부터 새 프리셋이 적용된다.  
10) 각 사진 보정값은 sidecar로 저장되며 앱 재실행 후에도 복원된다.  
11) Smart Export Pipeline이 동작해 종료 시점 대기 시간이 유의미하게 감소한다(목표: 10분 → 1~2분 체감).
