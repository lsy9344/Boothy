# 제품 요구사항 문서(PRD) v1.0 — (가칭) RapidTetherRAW

문서 버전: v1.0  
작성일: 2025-12-30  
대상 릴리스: MVP → 1.0  
작성자: Product Owner / Engineering

---

## 1. 개요(Executive Summary)
RapidTetherRAW는 **Canon EOS 700D 단일 기종**을 대상으로 하는 **Windows 전용 테더 촬영 + RAW 현상 + Export** 데스크톱 앱이다.  
핵심 목표는 Lightroom Classic에서 사용하던 작업 중 “**카메라 연결(테더) / 라이브뷰 / 촬영 후 즉시 표시 / 현상(보정) / 내보내기**”만을 **초미니멀 UX**로 제공하는 것이다.

본 제품은 **RapidRAW(AGPL-3.0) 코드를 그대로 활용하는 포크(fork)**를 전제로 한다. RAW 현상 엔진/보정 파라미터/프리셋/비파괴(사이드카) 워크플로우는 RapidRAW의 검증된 구현을 기반으로 하며, 테더링은 **Canon EDSDK(사용자 로컬 설치/경로 설정)**를 통해 추가 구현한다.

## 2. 제품 목표와 비전(Goals & Vision)
### 2.1 비전
스튜디오 현장에서 “촬영 → 즉시 확인(PC) → 동일 보정(필터/프리셋) 연속 적용 → 필요한 컷만 빠르게 Export”를 가장 적은 클릭으로 완료한다.

### 2.2 목표
- **EOS 700D 테더 워크플로우 최적화**: 연결 안정성, 라이브뷰, 촬영 후 즉시 반영(Preview).
- **보정은 빠르고 직관적**: RapidRAW 수준의 반응성과 현상 기능을 제공하되 UI는 최소 구성.
- **Lightroom Classic 사용 방식과 동일한 “캡처 프리셋” 개념**:
  - 사용자가 선택한 “캡처 프리셋(필터)”은 **이후 촬영분부터 계속 자동 적용**된다.
  - 프리셋을 바꾸면 **바뀐 이후 촬영분부터** 새로운 프리셋이 적용된다.
  - 각 사진은 개별적으로 보정값을 저장(비파괴)할 수 있다.

### 2.3 비목표(의도적으로 하지 않는 것)
- 다중 카메라/다중 제조사 지원
- 방대한 라이브러리/태깅/레이팅/검색/AI 기능/파노라마 등 Lightroom의 전체 기능
- EDSDK 바이너리/런타임을 앱에 동봉(재배포하지 않음)

## 3. 타겟 사용자 및 시나리오(Personas & Scenarios)
### 3.1 타겟 사용자
- 스튜디오/촬영 현장 사용자(운영자 1인): EOS 700D를 고정 세팅으로 쓰며, 촬영 직후 PC에서 확인/보정/내보내기를 반복한다.

### 3.2 핵심 시나리오(End-to-End)
1) 사용자는 세션 폴더를 지정한다(기본 제안: `YYYY-MM-DD-HH-MM`).  
2) 카메라(EOS 700D)를 USB로 연결하고 앱에서 “Connect”를 누른다.  
3) (필요 시) 라이브뷰 팝업 창을 열어 구도를 확인한다.  
4) 사용자는 “캡처 프리셋”을 선택한다.  
5) 물리 리모컨으로 촬영하거나 앱의 “Capture” 버튼을 눌러 촬영한다.  
6) 촬영된 RAW가 PC 세션 폴더로 전송되고, 앱은 **가장 최근 촬영본을 즉시 표시**하며 선택된 캡처 프리셋을 자동 적용한다.  
7) 사용자는 필요 시 슬라이더를 조정하고, 결과를 JPEG로 Export 한다(단일/선택 다중/세션 전체).

## 4. 제품 범위(Scope)
### 4.1 In-Scope (MVP/1.0)
#### 4.1.1 테더링(EDSDK)
- 지원 기종: **Canon EOS 700D만 지원**
- 연결/해제
- **라이브뷰**: 별도 “전용 팝업 윈도우”로 표시(메인 화면과 분리)
- 촬영 트리거:
  - **물리 리모컨 촬영 감지**(카메라에서 촬영된 이벤트 처리)
  - 앱 내 “Capture” 버튼으로 원격 촬영(보조 수단)
- 저장 정책: **PC(Host)로만 저장**
- 촬영 파일 수신 후 자동 반영:
  - 세션 폴더에 저장
  - 앱 UI에 즉시 로드(최근 촬영본 자동 선택)
  - 선택된 “캡처 프리셋” 자동 적용(이후 촬영분에도 유지)

#### 4.1.2 RAW 현상(보정)
RapidRAW의 현상 기능 중 아래 항목을 제공한다(“모두 포함”).
- 화이트밸런스: Temp/Tint
- 톤: Exposure/Contrast/Highlights/Shadows/Whites/Blacks
- 컬러: Vibrance/Saturation
- 디테일: Sharpening/Noise Reduction(휘도/색상)
- 변형: Crop/Rotate/Flip
- 톤 커브: Luma 및 RGB 커브(가능 범위는 RapidRAW 구현에 따름)
- 프리셋:
  - 캡처 프리셋(테더 촬영에 자동 적용되는 프리셋)
  - 사용자 프리셋 저장/불러오기/적용
- 히스토그램/클리핑 표시(가능하면, RapidRAW 기반)
- Undo/Redo(비파괴 편집)

#### 4.1.3 비파괴 저장(사이드카)
- 각 RAW 파일 옆에 **편집 값(sidecar)**을 저장한다(예: RapidRAW의 `.rrdata` 방식 또는 호환 포맷).
- 기존 사진을 다시 열면 저장된 편집 값이 복원된다.
- “캡처 프리셋”은 **새로 들어오는 촬영본의 초기 값**으로만 적용되며, 과거 촬영본의 값은 자동으로 바뀌지 않는다.

#### 4.1.4 내보내기(Export)
- 출력 포맷: JPEG(필수), 기타 포맷은 차기 고려(Out-of-Scope 참조)
- 메타데이터/ICC 처리: 가능한 범위 내에서 유지/임베드(구현은 RapidRAW 기반)
- 단일/다중 선택/세션 전체 Export
- 파일명 규칙(템플릿) 및 출력 폴더 선택

#### 4.1.5 세션 관리
- 세션 시작 시 사용자가 세션 폴더를 지정한다.
- 기본 제안 이름: `YYYY-MM-DD-HH-MM` (사용자 수정 가능)
- 세션 내 이미지 목록(최소 UI):
  - 최근 촬영 순 리스트/필름스트립 수준의 최소 탐색

### 4.2 Out-of-Scope (v1 비범위)
- 다중 카메라 지원 / 카메라 선택 UI
- 고급 라이브러리 기능(폴더 트리 관리, 태깅, 레이팅, 검색)
- 로컬 마스킹/브러시, AI 마스킹/생성형 편집, 파노라마/스티칭
- 렌즈 보정/퍼스펙티브 워핑 등 고급 변형
- Lightroom XMP 라운드트립(읽기/쓰기 호환)
- 클라우드 동기화/계정/구독 결제 기능

## 5. 사용자 경험(UX) 요구사항
### 5.1 핵심 UI 구성(초미니멀)
- 메인 창(항상 사용):
  - 상태 바: 연결 상태(Connected/Disconnected), 저장 경로, 최근 이벤트/에러
  - 캡처 프리셋 선택(드롭다운)
  - “Capture” 버튼
  - 최근 촬영/선택 이미지 프리뷰 + 현상 패널(슬라이더/커브)
  - Export 버튼
- 라이브뷰 팝업(필요 시만):
  - “Open Live View”로 열기/닫기
  - 연결 해제 시 자동 종료 또는 안내 표시

### 5.2 UX 원칙
- 촬영/수신/표시까지의 흐름은 **중단 없이 자동 진행**(사용자에게 “다음에 뭘 눌러야 하는지”를 최소화)
- 에러는 짧고 구체적으로(예: “EDSDK 경로 설정 필요”, “카메라 연결 끊김”, “저장 폴더 권한 없음”)
- 라이브뷰/촬영 동시성:
  - 촬영 순간 라이브뷰가 끊길 수 있음을 전제로 하고,
  - 촬영 후 **자동으로 라이브뷰 재개**(가능한 경우) 또는 재개 버튼을 제공

## 6. 기능 요구사항(Functional Requirements)
### 6.1 테더 연결/세션
- 세션 폴더 지정 없이 테더 연결을 시작할 수 없다.
- 연결 성공 시:
  - 저장 위치가 “PC(Host)”가 되도록 설정한다(EDSDK `SaveTo` 등).
  - 촬영 이벤트 수신을 위한 핸들러를 등록한다(Object/Property/State 이벤트).
- 연결 해제 시:
  - 핸들러/세션 정리, 라이브뷰 중이면 종료, UI 상태 초기화.

### 6.2 촬영(리모컨/앱 버튼)
- 물리 리모컨 촬영:
  - 촬영 이벤트를 수신하고, 생성된 파일을 PC로 전송/저장한다.
  - 저장 완료 후 앱이 자동으로 해당 파일을 로드한다.
- 앱 버튼 촬영:
  - 원격 릴리즈 명령으로 촬영을 트리거한다.
  - 이후 흐름은 동일(이벤트 수신/전송/로드).

### 6.3 라이브뷰(팝업)
- 팝업 창에서 프레임을 주기적으로 갱신한다.
- 네트워크/디바이스 오류 시:
  - 팝업에 “끊김/재시도/닫기” 상태를 표시한다.

### 6.4 이미지 로드/보정/저장
- 촬영된 RAW를 로드하면:
  - 해당 이미지에 기존 sidecar가 있으면 적용
  - 없으면 “현재 캡처 프리셋”으로 초기 보정값을 생성하고 sidecar를 만든다
- 사용자가 보정을 변경하면 자동 저장(또는 명시적 저장 옵션)한다.
- Undo/Redo는 이미지별 히스토리를 유지한다(구현 범위는 RapidRAW 기반).

### 6.5 프리셋
- 사용자 프리셋: 저장/불러오기/삭제
- 캡처 프리셋: “현재 선택값”을 유지하며 이후 촬영본에만 자동 적용

### 6.6 Export
- 내보내기 대상:
  - 현재 사진
  - 선택한 사진들
  - 세션 전체
- 내보내기 옵션:
  - 품질(quality), 긴 변 기준 리사이즈(선택), 메타데이터 유지(선택)
  - 출력 폴더 및 파일명 템플릿
- 내보내기 중 UI는 멈추지 않는다(백그라운드 작업).

## 7. 비기능 요구사항(Non-Functional Requirements)
### 7.1 성능
- 촬영 후 “최근 촬영본 표시” 목표: **2초 이내**(일반 케이스), 최악의 경우에도 사용자에게 진행 상태 표시
- 슬라이더 조정 시 프리뷰 반응: **지각 가능한 지연 최소화**(RapidRAW 수준 유지가 목표)
- 라이브뷰: 가능한 범위 내에서 **안정적 프레임 갱신**(기기/드라이버 영향 인정)

### 7.2 안정성/신뢰성
- 케이블 분리/절전/오류 시:
  - 명확한 상태 전환 + 재연결 동작 제공
- 촬영 이벤트 중복/누락 방지(파일 전송 완료 확인)
- 충돌 시:
  - 마지막 저장된 sidecar 기준으로 복구 가능해야 한다.

### 7.3 호환성
- OS: Windows 10/11 x64
- 카메라: Canon EOS 700D 단일
- EDSDK: 사용자가 로컬에 설치하고 앱에 경로를 설정한다(재배포하지 않음).

## 8. 기술/의존성(Tech & Dependencies)
### 8.1 기본 전략
- RapidRAW를 기반으로 포크하여, 불필요한 기능(UI/라이브러리/AI 등)을 제거 또는 비활성화한다.
- 테더링은 EDSDK를 **Rust FFI(또는 별도 네이티브 모듈)로 래핑**하여 통합한다.

### 8.2 주요 구성요소(개요)
- UI: Tauri + Web 프론트엔드(업스트림 유지)
- RAW 처리/현상: RapidRAW 엔진(업스트림 코드 그대로 활용)
- GPU 처리: wgpu/WGSL(업스트림)
- 테더: Canon EDSDK(프로프리어터리, 사용자 설치)

## 9. 라이선스/법적(Compliance)
- 제품 소스 공개: **AGPL-3.0 준수(포크 기반)**
- Canon EDSDK:
  - SDK 바이너리/헤더/런타임을 저장소/배포물에 포함하지 않는다.
  - 사용자가 로컬에 설치하고 경로를 설정하도록 안내한다.
- “Lightroom” 등 타사 상표/브랜드는 비교 설명 수준을 넘는 사용을 금지한다.

## 12. 리스크 및 완화(Risks & Mitigations)
- **EDSDK EULA/배포 제약**: “사용자 설치 + 경로 설정”을 기본으로 하고 문서/UX를 강화
- **라이브뷰/촬영 동작의 바디 특성**: 촬영 시 라이브뷰 끊김/재개를 UX로 흡수(자동 재개/명확한 상태 표시)
- **업스트림 변화/유지보수**: RapidRAW 특정 릴리스/커밋에 핀(pin)하고, 기능 추가는 최소한으로 유지

## 13. 수용 기준(Acceptance Criteria)
1) Windows에서 EOS 700D 연결/해제/재연결이 안정적으로 동작한다.  
2) 물리 리모컨 촬영 시, RAW 파일이 PC 세션 폴더로 저장되고 앱에 자동으로 표시된다.  
3) “캡처 프리셋”을 선택한 뒤 촬영하면, 이후 촬영분에 동일 프리셋이 자동 적용되고, 프리셋 변경 이후 촬영분부터 새 프리셋이 적용된다.  
4) 각 사진의 보정값은 sidecar로 저장되며, 앱 재실행 후에도 동일하게 복원된다.  
5) 사용자는 현재/선택/세션 전체를 JPEG로 Export 할 수 있다.
