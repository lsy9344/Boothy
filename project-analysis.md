# Project Analysis — RapidRAW (Upstream) + Boothy (Fork Context)

## Context

- Workspace project: **Boothy (RapidTetherRAW)** — a fork target built on top of **RapidRAW**.
- Upstream repo: `https://github.com/CyberTimon/RapidRAW` (cloned to `upstream/RapidRAW` at commit `a931728`).
- Provided requirements: `prd-rapidraw-customer-mode-v1.1.1.md` (Customer/Admin modes, kiosk workflow, ExportLock, privacy reset, Windows-only EOS 700D tethering via EDSDK).

## What This Repo Actually Is (Upstream Reality)

RapidRAW is a Tauri 2 desktop app with:

- Frontend: React + TypeScript + Vite + Tailwind (`upstream/RapidRAW/src/App.tsx` is the orchestration hub)
- Backend: Rust (Tauri commands + GPU processing + RAW develop + AI features)
- Non-destructive edits: sidecar files stored next to images (`*.rrdata`)
- Persistent app data: settings + presets stored in Tauri `app_data_dir`
- Cache: thumbnails stored in Tauri `app_cache_dir/thumbnails`

Reference doc generated by `*document-project`: `docs/brownfield-architecture.md`

## Key Architectural Hotspots for the PRD

### 1) Mode & Flow Orchestration (Customer/Admin)

- `upstream/RapidRAW/src/App.tsx` currently drives:
  - `activeView` switching (library vs community)
  - editor vs library view (based on `selectedImage`)
  - global event listeners from Rust (`listen(...)`) for preview, import/export, indexing, etc.

For “Customer Mode” (railroaded state machine), this file is the primary integration point.

### 2) Backend Command Surface (Invoke Contract)

- TS command names are centralized in `upstream/RapidRAW/src/components/ui/AppProperties.tsx` (`enum Invokes`).
- Rust registers commands in `upstream/RapidRAW/src-tauri/src/main.rs` via `tauri::generate_handler![...]`.

Any fork changes should treat this as an API contract.

### 3) Session Reset / Privacy Reset

Observed building blocks (upstream capability):

- Sidecars: `*.rrdata` next to images (there is a delete helper: `file_management::clear_all_sidecars(root_path)`).
- Thumbnails: `app_cache_dir/thumbnails` (delete via `file_management::clear_thumbnail_cache(app_handle)`).
- Background tasks: `export_task_handle`, `indexing_task_handle` inside `AppState` (need explicit cancellation semantics for kiosk reset).

Confirmed requirement (Boothy): **reset is “app state/cache only”** (do not delete session folder files). This implies:

- Clear in-memory UI state (selection, navigation, current session state machine).
- Clear thumbnail cache and other transient cache/temporary files.
- Cancel/abort long-running background tasks cleanly.

### 4) ExportLock Gate

Export is implemented in `upstream/RapidRAW/src-tauri/src/main.rs` with:

- `export_image`, `batch_export_images`, `cancel_export`, `estimate_export_size`
- Events to UI: `batch-export-progress`, `export-complete`, `export-error`, `export-complete-with-errors`

Customer Mode’s “ExportLock” can be implemented at the UI orchestration level by gating transitions until `export-complete` (and handling error/cancel paths).

Confirmed output naming requirement (Boothy):

- Export filename pattern: `예약ID-{hh}시-{sequence}` (example: `R12345-14시-001.jpg`)
- Upstream supports `{hh}` and `{sequence}` placeholders via `file_management::generate_filename_from_template(...)`; reservation ID can be injected by constructing the template at runtime.

### 5) Tethering (Not Present Upstream)

Upstream RapidRAW is file-based; Canon EDSDK tethering does not exist in this codebase.

Fork impact (Boothy):

- Introduce a new backend module for camera control + capture ingestion (Windows-only).
- Add new invokes/events for:
  - camera connect/disconnect state
  - live view (optional)
  - capture trigger and file arrival
  - error reporting + recovery

Confirmed capture ingestion requirement (Boothy):

- Captures are saved **directly into the active session folder** (not staged/imported).
- Customer UI should be restricted to the session folder (no free navigation).

## Notable Gotchas Found During Verification

1. `upstream/RapidRAW/index.html` references `/src/main.jsx`, but the repo entrypoint is `src/main.tsx` (verify dev/build behavior before forking).
2. UI listens for `export-cancelled`, but backend cancellation currently aborts the task without clearly emitting that event (verify actual runtime behavior).

## Confirmed Decisions (From Latest Clarifications)

1. Customer Mode file navigation: **limited to the session folder**
2. Privacy reset: **clear app state/cache only** (do not delete session folder files)
3. Export naming: **`예약ID-{hh}시-{sequence}`**
4. EDSDK capture ingestion: **save directly into the session folder**
